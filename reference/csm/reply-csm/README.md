# reply-csm · 사용 가이드

> 캠프 학습용 스킬. 같은 고객 이메일을 같은 스킬로 답하지만, 내가 만드는 환경이 두꺼워질수록 답변이 발전하는 걸 직접 본다.

---

## 1. 설치

이 폴더 자체를 **본인 Claude Code 작업 환경의 스킬 폴더로 복사**한다.

```bash
# 캠프 repo 루트에서
cp -R teams/csm/templates/reply-csm ~/.claude/skills/reply-csm
# 또는 프로젝트 단위로 쓰려면
mkdir -p .claude/skills && cp -R teams/csm/templates/reply-csm .claude/skills/
```

설치 확인:
```bash
ls ~/.claude/skills/reply-csm/SKILL.md
```

---

## 2. 호출 방법

Claude Code 세션에서:

### Reply 모드 (답장 생성)

```
"reply-csm 써줘. 메일 붙여넣을게:

[고객 이메일 본문]"
```

또는 단순히:

```
"이 고객 메일 답장 좀 써줘:
[고객 이메일 본문]"
```

(스킬이 자동으로 매칭됨)

### Review 모드 (처음 ↔ 지금 비교)

```
"reply-csm 리뷰하자"
"내가 답변한 거 처음이랑 지금 비교해줘"
```

## 5. 가드레일 (운영진 메모)

- **본문 전체 저장 X**: 메모리에는 200자 preview만. 본문은 휘발.
- **환각 금지**: MCP 응답 없으면 모드 2 수준으로 fallback. 가짜 일정·이름 만들지 말 것.
- **금지어 준수**: 모드 2·3에서 CLAUDE.md 금지어 절대 사용 X.
- **본인 진짜 메일 사용**: 운영진 샘플 X. 본인 최근 고객 메일 1통 가져오기.

## 6. 트러블슈팅

| 증상 | 원인 | 대응 |
|------|------|------|
| 모드가 자동 감지 안 됨 | cwd가 잘못됨 (CLAUDE.md 다른 폴더에 있음) | `pwd`로 확인 후 올바른 위치에서 `claude` 실행 |
| 모드 3인데 모드 2 결과 | MCP 권한 만료 | claude.ai에서 재인증 → Claude Code 재시작 |

| 가짜 일정·이름 나옴 | 모드 3 환각 (드물지만 가능) | 운영진에 보고. 가드레일 강화 검토 |
