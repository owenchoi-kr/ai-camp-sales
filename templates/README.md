# Day 3 스킬 템플릿 4종

> Day 3 실습 재료. 이 중 **1개를 골라** 내 딜 맥락으로 3요소만 수정(0.5 → 1)한 뒤,
> 그 폴더를 `.claude/skills/` 로 옮기면 "내 스킬"이 된다.

## 어떤 걸 고를까

| 템플릿 | 언제 쓰나 | 입력 → 산출 |
|--------|-----------|-------------|
| **call-prep** | 미팅 들어가기 전 브리핑이 필요할 때 | 계정명·미팅 성격·아젠다 → 콜 프렙 브리핑 |
| **battlecard** | 경쟁 MMP 계정을 이관 설득해야 할 때 | 계정명·경쟁사·갱신 시기 → 이관 배틀카드 |
| **meeting-followup** | 미팅 끝나고 후속 처리할 때 | 미팅노트 → 액션아이템·CRM 제안·메일 초안(국/영) |
| **draft-outreach** | 새 프로스펙트에 처음 닿을 때 | 회사·역할·접점 계기 → 콜드/웜 메일 2버전 |

## 공통 구조: 모드 자동 감지

4종 모두 환경을 감지해 모드 1·2·3으로 분기한다. **같은 입력이라도 환경이 두꺼워질수록 결과가 좋아진다.**

- **모드 1** — CLAUDE.md 없음. 입력한 정보만 사용 (순정 Claude).
- **모드 2** — CLAUDE.md 있음. Day 1에서 만든 내 세그먼트·톤·용어가 반영된다.
- **모드 3** — CLAUDE.md + 도구(MCP/스킬). 계정 히스토리 조회·초안 draft 저장까지.

Day 3 실습에서는 보통 모드 2~3을 체감하게 된다.

## 커스터마이징 방법 (0.5 → 1)

1. 템플릿 폴더를 하나 고른다.
2. `SKILL.md` 맨 위 주석의 안내에 따라 `{{ }}` 로 표시된 슬롯(2~3개)을 내 맥락으로 바꾼다.
3. 폴더째 `.claude/skills/` 로 옮긴다.
4. Claude Code에서 트리거 문구(예: "콜프렙", "미팅 팔로업")로 호출해 본다.

## 가드레일 (전 템플릿 공통)

- 확인되지 않은 사실(날짜·이름·이력·수치)을 지어내지 않는다. 못 구하면 "미확보"로 정직하게 둔다.
- 고객 메일은 **초안까지만**. 발송은 항상 사람이 확인 후 직접 한다.
- 경쟁사 비방 금지 — 사실 기반 비교만 (battlecard).
- CLAUDE.md에 넣은 금지어는 어떤 산출물에도 쓰지 않는다.
