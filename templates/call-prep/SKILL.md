# call-prep

> 미팅 카드 하나를 콜 프렙 브리핑으로 바꾼다. 같은 입력이라도 CLAUDE.md·계정 히스토리 소스가 붙을수록 브리핑이 깊어진다.

## 1. 입력 받기

다음을 받는다. 없으면 짧게 묻는다.

- **계정명** (필수)
- **미팅 성격**: 신규 / 갱신 / QBR 중 하나
- **아젠다** (한 줄이라도)
- (선택) 아는 배경 — 붙여넣기

## 2. 환경 자동 감지 (모드 결정)

1. **CLAUDE.md 있는가?** — cwd 또는 부모 디렉토리에서 `ls CLAUDE.md ../CLAUDE.md 2>/dev/null`
2. **계정 히스토리 소스가 있는가?** — `my-account-sync` 스킬(Day 2 산출물)이 설치돼 있거나, Gmail/Circleback/Notion/Slack MCP가 연결돼 있는가?

| 조건 | 모드 |
|------|------|
| CLAUDE.md 없음 | 모드 1 |
| CLAUDE.md 있음 + 히스토리 소스 없음 | 모드 2 |
| CLAUDE.md 있음 + 히스토리 소스 1개+ | 모드 3 |

## 3. 계정 히스토리 수집

- `my-account-sync` 스킬이 있으면 **그 스킬을 호출**해 계정 히스토리 브리핑을 받는다.
- 없으면 사용자에게 요청: "이 계정 관련 지난 메일/미팅노트/내부 논의가 있으면 붙여넣어 주세요. 없으면 그냥 진행할게요."
- 붙여넣기도 없으면 히스토리 없이 진행하고, 브리핑에 "히스토리 미확보" 라고 명시한다.

## 4. 모드별 동작

**모드 1 (CLAUDE.md 없음)** — 입력한 정보만 사용. 일반적인 세일즈 프렙 프레임으로 작성하고, 계정 특이사항·이력은 추측하지 않는다.

**모드 2 (CLAUDE.md 있음)** — `CLAUDE.md`를 읽어 {{담당_세그먼트_특성}}·{{내_톤}}·{{딜_스테이지_용어}}를 반영한다. 예상 질문·반론을 이 세그먼트에서 실제로 나오는 것으로 좁힌다. 다음 스텝은 우리 팀 파이프라인 단계 이름({{딜_스테이지_용어}})으로 제안한다.

**모드 3 (CLAUDE.md + 히스토리 소스)** — 위에 더해, 수집한 히스토리에서 사실만 인용해 브리핑을 보강한다. 지난 논의·미해결 이슈·담당자를 근거와 함께 붙인다. 조회 실패 시 그 부분만 모드 2 수준으로 fallback.

## 5. 출력 형식

```
🧭 모드 N · 활용된 맥락: [계정 히스토리 / CLAUDE.md / ...]

# 콜 프렙 — {계정명} ({미팅 성격})

## 미팅 목표
- 이 미팅에서 반드시 얻을 것 1~2개

## 계정 히스토리 요약
- 3~5줄. 히스토리 없으면 "미확보"

## 예상 질문·반론 Top 3
1. [질문/반론] → [대응]
2. ...
3. ...

## 오프닝 제안
- 첫 2~3문장 스크립트

## 다음 스텝 제안
- {{딜_스테이지_용어}} 기준으로 한 단계 전진시키는 액션
```

## 가드레일

- **모든 사실 주장에는 출처(URL) 또는 `[가설]` 라벨을 단다.** 조회·붙여넣기로 확인된 것은 출처를, 추론한 것은 `[가설]`을 붙인다. 라벨 없는 단정 금지. (거짓 레퍼런스가 실제 영업 사고로 이어진 사례가 있다.)
- 히스토리를 못 구했으면 "미확보"라고 정직하게 쓴다. 그럴듯하게 채우지 않는다.
- 고객사·담당자 실명은 입력받은 그대로만 쓰고, 예시로 지어내지 않는다.
