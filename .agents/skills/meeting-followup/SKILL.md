# meeting-followup

> 미팅노트 하나를 액션아이템·CRM 업데이트·고객 메일 초안으로 바꾼다. 메일은 초안까지만 — 발송은 반드시 사람이 한다.

## 0. Codex 실행 노트

- description 매칭으로 자동 호출되거나 `$meeting-followup` 로 명시 호출.
- 컨텍스트 파일은 **AGENTS.md**.
- 미팅노트 조회는 `config.toml` 의 Circleback/Notion MCP, 메일 draft 저장은 Gmail MCP. 없으면 붙여넣기·텍스트 출력으로 fallback.

## 1. 입력 받기 (미팅노트)

1. **Circleback/Notion MCP 연결됨?** — 연결돼 있으면 최근 미팅을 조회하거나, 사용자가 준 검색어/날짜로 해당 미팅노트를 가져온다.
2. 연결 안 됐거나 못 찾으면 요청: "미팅노트를 붙여넣어 주세요."

## 2. 환경 자동 감지 (모드 결정)

1. **AGENTS.md 있는가?** — `ls AGENTS.md ../AGENTS.md 2>/dev/null`
2. **Gmail MCP 연결됨?** — 메일 초안을 draft로 저장할 수 있는가?

| 조건 | 모드 |
|------|------|
| AGENTS.md 없음 | 모드 1 |
| AGENTS.md 있음 + Gmail MCP 없음 | 모드 2 |
| AGENTS.md 있음 + Gmail MCP 있음 | 모드 3 |

## 3. 산출물 3종 (모든 모드 공통)

**① 핵심 액션아이템**

| 액션 | 담당 | 기한 |
|------|------|------|
| ... | ... | 미정이면 {{팔로업_기한_기본값}} |

**② CRM 업데이트 제안** — 스테이지 이동 제안 + 다음 스텝. (제안만. CRM 쓰기는 사람이 확인 후.)

**③ 고객 팔로업 메일 초안** — 국문 + 영문 각 1개. 말투·서명은 {{내_톤_시그니처}}.

## 4. 모드별 차이

**모드 1** — 미팅노트 내용만으로 3종 작성. 톤은 일반적인 정중한 SaaS 톤.

**모드 2** — `AGENTS.md`를 읽어 {{내_톤_시그니처}}와 금지어를 반영한다. CRM 업데이트 제안은 우리 팀 스테이지 용어로 쓴다.

**모드 3** — 위에 더해, 완성된 메일 초안(국/영문)을 **Gmail MCP draft로 저장**한다. 저장 후 "초안 저장됨 — 확인 후 직접 발송하세요"라고 안내한다.

## 5. 출력 형식

```
🧭 모드 N · 활용된 맥락: [Circleback / AGENTS.md / Gmail draft / ...]

# 미팅 팔로업 — {계정/미팅명}

## ① 액션아이템
[표]

## ② CRM 업데이트 제안
- 스테이지: A → B / 다음 스텝: ...

## ③ 고객 팔로업 메일 초안
### 국문
[본문 + {{내_톤_시그니처}}]
### English
[본문 + signature]

[모드 3] — Gmail draft 저장됨. 발송은 직접 확인 후 진행하세요.
```

## 가드레일

- **오탈자 검수 1회 강제.** 메일 초안(국/영문)을 만든 뒤, draft 저장·"완료" 보고 **전에** 회사명·인명·제품명을 한 번 재검토한다. 전사 기반 미팅노트는 고유명사 오인식이 흔하다 — 원본 입력과 대조해 바로잡는다. 검수 전 "완료" 보고 금지.
- **메일 자동 발송 금지.** 초안 생성/draft 저장까지만. 발송 버튼은 항상 사람이 누른다.
- 미팅노트에 없는 약속·수치·일정을 지어내지 않는다. 노트 근거만 쓴다.
- 액션아이템 담당·기한이 노트에 없으면 "미정({{팔로업_기한_기본값}})"으로 표시하고 확정을 요청한다.
- AGENTS.md 금지어는 메일 초안에 절대 쓰지 않는다.
