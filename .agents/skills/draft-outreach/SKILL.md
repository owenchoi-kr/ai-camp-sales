# draft-outreach

> 프로스펙트 하나를 콜드/웜 아웃리치 메일 초안 2버전으로 바꾼다. 발송은 반드시 사람이 한다.

## 0. Codex 실행 노트

- description 매칭으로 자동 호출되거나 `$draft-outreach` 로 명시 호출.
- 컨텍스트 파일은 **AGENTS.md**.
- Gmail draft 저장은 `~/.codex/config.toml` 의 `[mcp_servers.gmail]` 로 붙인 Gmail MCP 서버의 draft 생성 도구로 한다. 없으면 텍스트 초안까지만.

## 1. 입력 받기

- **회사** (필수)
- **담당자 역할** — 직함/팀
- **접점 계기** — 콜드인지 웜인지, 어떤 계기로 닿는지 (이벤트·소개·다운로드 등)

없으면 짧게 묻는다. 접점 계기가 있으면 웜, 없으면 콜드로 톤을 잡는다.

## 2. 환경 자동 감지 (모드 결정)

1. **AGENTS.md 있는가?** — `ls AGENTS.md ../AGENTS.md 2>/dev/null`
2. **Gmail MCP 연결됨?** — 초안을 draft로 저장할 수 있는가? (`/mcp` 로 확인)

| 조건 | 모드 |
|------|------|
| AGENTS.md 없음 | 모드 1 |
| AGENTS.md 있음 + Gmail MCP 없음 | 모드 2 |
| AGENTS.md 있음 + Gmail MCP 있음 | 모드 3 |

## 3. 모드별 동작

**모드 1 (AGENTS.md 없음)** — 입력 정보만으로 일반적인 아웃리치 2버전을 작성한다. 밸류 프롭·톤은 무난한 기본값으로.

**모드 2 (AGENTS.md 있음)** — `AGENTS.md`를 읽어 {{밸류_프롭_한줄}}을 훅에 녹이고, {{타깃_페르소나}}에 맞게 관심사를 조준하며, 내 톤·시그니처를 반영한다. {{금지어}}는 절대 넣지 않는다.

**모드 3 (AGENTS.md + Gmail MCP)** — 위에 더해, 완성된 두 버전을 **Gmail MCP draft로 저장**한다. 저장 후 "초안 저장됨 — 확인 후 직접 발송하세요"라고 안내한다.

## 4. 출력 형식 (2버전)

```
🧭 모드 N · 활용된 맥락: [AGENTS.md / Gmail draft / ...]

# 아웃리치 초안 — {회사} / {담당자 역할} ({콜드|웜})

## 버전 A — 짧은 버전 (3~5문장)
제목: ...
본문: [훅 → {{밸류_프롭_한줄}} → 가벼운 CTA]

## 버전 B — 상세 버전
제목: ...
본문: [훅 → 맥락/근거 → {{밸류_프롭_한줄}} → 구체 CTA + 시그니처]

[모드 3] — Gmail draft 2건 저장됨. 발송은 직접 확인 후 진행하세요.
```

## 가드레일

- **오탈자 검수 1회 강제.** 두 버전을 만든 뒤, draft 저장·"완료" 보고 **전에** 회사명·담당자명·제품명을 한 번 재검토한다. 특히 음성/전사나 급하게 받아적은 입력은 고유명사 오인식이 흔하다 — 원본 입력과 대조해 바로잡는다. 검수 전 "완료" 보고 금지.
- **메일 자동 발송 금지.** 초안 생성/draft 저장까지만. 발송은 항상 사람이.
- 프로스펙트에 대해 확인되지 않은 사실(투자·매출·인물 이력)을 지어내지 않는다. 접점 계기로 준 것만 근거로 쓴다.
- {{금지어}}와 AGENTS.md 금지어는 두 버전 모두에서 쓰지 않는다.
- 과장·허위 약속("무조건", "100%") 금지. 검증 가능한 밸류만.
