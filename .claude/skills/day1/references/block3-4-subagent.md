## EXPLAIN

> **Phase A 시작 시 반드시 아래 형태로 출력한다:**
> ```
> 📖 공식 문서: https://code.claude.com/docs/ko/sub-agents

| 항목 | 내용 |
|------|------|
| 근본 원리 | **프로세스 격리 + Blank Slate** — Subagent는 새 Claude 프로세스를 띄운다. 메인 대화 컨텍스트를 물려받지 않고 빈 상태에서 시작해, 작업만 하고 요약만 돌려준다 |
| 비유 | 부하 직원 — 독립된 공간에서 특정 작업을 전담 처리 |
| 세일즈 예시 | "이 계정 정보를 Circleback·Notion·Slack에서 **동시에** 긁어와서 정리해줘" — 각 소스를 별도 에이전트가 병렬로 수집 |

```
┌─ 메인 Claude ─────────────────────────┐
│  대화 컨텍스트: [A, B, C, D...]        │
│  "계정 히스토리 모아줘" ──┐            │
│                          ▼            │
│  ┌─ Subagent ─────────────┐           │
│  │ 컨텍스트: [빈 상태]      │           │
│  │ 작업: 미팅노트만 수집    │           │
│  │ 결과: 요약 3줄 ─────────┼──▶ 전달   │
│  └────────────────────────┘           │
└───────────────────────────────────────┘
```

Day 2에 만드는 계정 리서치 스킬이 바로 이 구조다. Gmail·Circleback·Notion·Slack 4소스를 4개 에이전트가 **병렬로** 긁어와 하나의 브리핑으로 합친다. 그래서 계정명 하나만 넣어도 히스토리가 나온다.
