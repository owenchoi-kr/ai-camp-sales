# Block 4: 내 CLAUDE.md 세팅 — 오늘의 산출물

> 오늘 하루의 목적지. Claude가 참가자를 인터뷰하며 CLAUDE.md를 함께 채우고, **실제 파일로 저장**한다.
> 이 블록만 STOP PROTOCOL을 조금 다르게 쓴다: Phase A는 설명 + 인터뷰 예고 후 Stop, Phase B에서 대화형으로 인터뷰를 진행한다(AskUserQuestion + 자유서술 혼용, 그리고 Write로 파일 생성).

## EXPLAIN (Phase A)

> **Phase A 시작 시 반드시 아래 형태로 출력한다:**
> ```
> 📖 공식 문서: https://code.claude.com/docs/ko/memory
> ```

한 문단으로 개념을 다시 짚는다:

> CLAUDE.md는 Claude가 **매 대화를 시작할 때 자동으로 읽는 고정 문서**다. 담당 계정·내 톤·금지어를 여기 한 번 박아두면, 앞으로 콜 프렙이든 팔로업 초안이든 매번 "내 담당은 커머스고, 톤은 담백하게…"를 다시 설명할 필요가 없다. Block 1에서 밋밋했던 그 브리핑이, 이 파일 한 장 위에서 시작하면 달라진다.

이제 인터뷰할 **4가지**를 예고한다:

1. **담당 계정 / 세그먼트** — 어떤 계정을 맡는지, 버티컬(커머스·핀테크·게임 등)과 규모(엔터프라이즈·SMB)
2. **내 아웃리치 톤 / 시그니처** — 고객 앞 언어 스타일, 메일 서명, 자주 쓰는 표현
3. **금지어 / 주의사항** — 절대 쓰면 안 되는 표현 (예: 경쟁사 비방, 확정 안 된 할인·약속 단정)
4. **자주 하는 작업 3개** — 반복하는 업무 (예: 콜 프렙, 미팅 팔로업, 경쟁사 배틀카드)

Phase A 종료 문구:

```
---
👆 4가지를 인터뷰하면서 여러분의 CLAUDE.md를 같이 채웁니다.
준비되면 "시작"이라고 입력해주세요.
```

여기서 Stop한다. (Phase A에서는 AskUserQuestion·Write를 호출하지 않는다.)

## EXECUTE (Phase B) — 인터뷰 진행 규칙

사용자가 "시작"이라고 하면, 아래 순서로 **한 번에 한 주제씩** 인터뷰한다. 4개를 한꺼번에 쏟지 말고, 대화로 하나씩 끌어낸다.

### 인터뷰 원칙

- **선택지가 명확한 건 AskUserQuestion, 서술이 필요한 건 자유서술로.** (계정명·톤·서명은 자유서술, 세그먼트 유형은 AskUserQuestion이 편하다.)
- 한 주제당 답이 얇으면 **한 번만 더 캐묻는다** (예: "엔터프라이즈"라고만 하면 → "연 단위 계약인가요? 주로 어떤 이슈로 만나나요?"). 단, 취조하듯 물고 늘어지지 않는다. 0.5판이 목표다.
- 고객사 실명이 부담되면 "글로벌 커머스 앱 A사"처럼 익명으로 적어도 된다고 미리 알려준다.
- **확정 안 된 할인/약속을 금지어에 넣도록 유도**한다(회사 정책과 직결).

### 주제 1 — 담당 계정 / 세그먼트

먼저 AskUserQuestion으로 주력 세그먼트를 고르게 한다:

```json
AskUserQuestion({
  "questions": [{
    "question": "주로 담당하는 계정은 어느 쪽인가요? (복수 선택 가능)",
    "header": "담당 세그먼트",
    "options": [
      {"label": "커머스 / 리테일", "description": "쇼핑·이커머스 앱"},
      {"label": "핀테크 / 금융", "description": "은행·증권·페이"},
      {"label": "게임 / 엔터테인먼트", "description": "게임·콘텐츠·미디어"},
      {"label": "기타 (직접 입력)", "description": "위에 없는 버티컬"}
    ],
    "multiSelect": true
  }]
})
```

이어서 자유서술로 묻는다: "대표 계정 2~3개만 이름(또는 익명)으로 알려주세요. 규모(엔터프라이즈/SMB)와 주로 어떤 이슈로 만나는지도 한 줄씩."

### 주제 2 — 내 아웃리치 톤 / 시그니처

자유서술로 묻는다:
- "고객한테 메일 쓸 때 톤은 어떤가요? (예: 짧고 담백 / 따뜻하고 관계 중심 / 데이터로 설득)"
- "메일 서명(시그니처)은 어떻게 쓰세요? 자주 쓰는 오프닝·클로징 표현이 있나요?"

### 주제 3 — 금지어 / 주의사항

자유서술 + 유도:
- "고객 앞에서 절대 쓰면 안 되는 표현이 있나요?"
- 참가자가 막히면 예시를 던진다: "예를 들면 — 경쟁사(AppsFlyer/Adjust 등) 비방, '반드시'·'100% 보장' 같은 약속 단정, **아직 확정 안 된 할인·프로모션 언급**, 내부 용어 외부 노출. 이 중 해당되는 것 골라도 돼요."

### 주제 4 — 자주 하는 작업 3개

AskUserQuestion으로 고르게 하고, 부족하면 자유서술로 보완:

```json
AskUserQuestion({
  "questions": [{
    "question": "매주 반복하는 작업 3개를 골라주세요 (복수 선택)",
    "header": "반복 업무",
    "options": [
      {"label": "콜 프렙 / 미팅 준비", "description": "미팅 전 계정 히스토리 정리"},
      {"label": "미팅 팔로업", "description": "미팅 후 액션아이템·팔로업 메일"},
      {"label": "경쟁사 배틀카드", "description": "이관 설득 포인트 정리"},
      {"label": "아웃리치 / 콜드메일", "description": "신규 프로스펙트 접촉"}
    ],
    "multiSelect": true
  }]
})
```

### 파일 작성 — 반드시 실제로 저장한다

4주제 인터뷰가 끝나면:

1. 수집한 내용을 아래 구조의 마크다운으로 정리한다.
2. **Write 도구로 프로젝트 루트에 `CLAUDE.md`를 저장한다.** (경로: `ai-camp-sales/CLAUDE.md`)
   - 이미 `CLAUDE.md`가 있으면 덮어쓰지 말고, 참가자에게 "기존 파일이 있는데 이어붙일까요, 새로 만들까요?"라고 묻는다.
3. 저장 후 파일 내용을 그대로 보여주고, "이제 매 세션에서 이게 자동으로 읽힙니다"라고 안내한다.

작성 템플릿:

```markdown
# 내 세일즈 프로필 (CLAUDE.md)

## 담당 계정 / 세그먼트
- 주력 세그먼트: {버티컬}
- 대표 계정: {이름/익명 + 규모 + 주요 이슈}

## 내 아웃리치 톤 / 시그니처
- 톤: {짧고 담백 / 관계 중심 / 데이터 설득 등}
- 시그니처: {서명}
- 자주 쓰는 표현: {오프닝·클로징}

## 금지어 / 주의사항
- {경쟁사 비방 금지}
- {확정 안 된 할인·프로모션 언급 금지}
- {약속 단정("반드시"·"100% 보장") 금지}
- {기타}

## 자주 하는 작업
1. {작업1}
2. {작업2}
3. {작업3}
```

> 완벽할 필요 없다. **0.5판**이 목표다. 나머지는 2주차 프로젝트에서 실무로 채운다.

## QUIZ

```json

    ],

  }]

```
