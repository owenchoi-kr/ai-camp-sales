# Day 3: Clarify + 0.5→1 커스터마이징

이 스킬이 호출되면 아래 **STOP PROTOCOL**을 반드시 따른다.

두 가지 용도로 쓰인다:
- **라이브 수업(90분)**: 운영진과 함께 진행하며 실습 가이드로 사용
- **결석자 self-paced**: 혼자 `/day3`을 돌리면 Claude가 1:1로 그날 수업을 통째로 진행

> 오늘의 목적지는 **Block 3에서 템플릿 하나를 내 맥락으로 수정해 "동작하는 내 스킬 1개"를 만드는 것**이다.
> 관통 메시지: **"일 잘 시키는 사람이 AI도 잘 시킨다."** 모호하게 시키면 모호한 결과가 나온다.

---

## 용어 정리

| 용어 | 설명 |
|------|------|
| **Clarify** | 실행하기 전에 "정확히 뭘 원하는지"를 먼저 되묻는 단계. 머릿속 그림을 말로 꺼내는 과정 |
| **GIGO** | Garbage In, Garbage Out — 쓰레기가 들어가면 쓰레기가 나온다. 모호한 입력 → 모호한 결과 |
| **0.5 → 1** | 스킬을 0에서 만들지 않는다. 운영진 템플릿(0.5)을 받아 내 딜 맥락 3곳만 고쳐 1로 만든다 |
| **슬롯(`{{슬롯}}`)** | 템플릿에서 "여기를 내 맥락으로 바꾸세요"라고 표시된 빈칸. 각 템플릿에 3개 |
| **AskUserQuestion** | Claude가 열린 질문 대신 가설을 선택지로 제시하는 도구. 인지 부하를 낮춘다 |

## STOP PROTOCOL — 절대 위반 금지

> 이 프로토콜은 이 스킬의 최우선 규칙이다. 위반하면 수업이 망가진다.

### 각 블록은 반드시 2턴에 걸쳐 진행한다

```
┌─ Phase A (첫 번째 턴) ──────────────────────────────┐
│ 1. references/에서 해당 블록 파일의 EXPLAIN 섹션을 읽는다    │
│ 2. 개념을 설명한다                                        │
│ 3. references/에서 해당 블록 파일의 EXECUTE 섹션을 읽는다    │
│ 4. "지금 직접 실행해보세요"라고 안내한다                     │
│ 5. ⛔ 여기서 반드시 STOP. 턴을 종료한다.                    │
│                                                          │
│ ❌ 절대 하지 않는 것: 퀴즈 출제, QUIZ 섹션 읽기             │
│ ❌ 절대 하지 않는 것: AskUserQuestion 호출                  │
│ ❌ 절대 하지 않는 것: "실행해봤나요?" 질문                   │
└──────────────────────────────────────────────────────────┘

  ⬇️ 사용자가 돌아와서 "했어", "완료", "다음" 등을 입력한다

┌─ Phase B (두 번째 턴) ──────────────────────────────┐
│ 1. references/에서 해당 블록 파일의 QUIZ 섹션을 읽는다       │
│ 2. AskUserQuestion으로 퀴즈를 출제한다                     │
│ 3. 정답/오답 피드백을 준다                                 │
│ 4. 다음 블록으로 이동할지 AskUserQuestion으로 묻는다         │
│ 5. ⛔ 다음 블록을 시작하면 다시 Phase A부터.                │
└──────────────────────────────────────────────────────────┘
```

### 핵심 금지 사항 (절대 위반 금지)

1. **Phase A에서 AskUserQuestion을 호출하지 않는다** — 설명 + 실행 안내 후 바로 Stop
2. **Phase A에서 퀴즈를 내지 않는다** — QUIZ 섹션은 Phase B에서만 읽는다
3. **Phase A에서 "실행해봤나요?"를 묻지 않는다** — 사용자가 먼저 말할 때까지 기다린다
4. **한 턴에 EXPLAIN + QUIZ를 동시에 하지 않는다** — 반드시 2턴으로 나눈다

### 공식 문서 URL 출력 (Block 1~4, 절대 누락 금지)

Block 1~4의 Phase A 시작 시, 해당 reference 파일 상단의 `> 공식 문서:` URL을 **그대로 출력**한다.

```
📖 공식 문서: [URL]
```

- URL이 여러 개면 전부 출력한다. 요약·생략하지 않는다.
- Block 0(회고)·Block 5(마무리)는 공식 문서 출력을 생략한다.

### 정확성 가드레일

Claude Code 기능 질문이 오면 claude-code-guide 에이전트(내장 도구)로 답한다. 내장 답변이 부정확하다고 판단되면, 공식 문서를 `curl`로 파일에 저장한 뒤 Read로 꼼꼼히 읽고 다시 답한다(WebFetch는 요약/손실 위험이 있어 쓰지 않는다). **기능 설명이 불확실하면 지어내지 말고 reference 파일의 설명 범위 안에서만 답한다.**

### Phase A 종료 시 필수 문구

Phase A 마지막에는 반드시 아래 문구를 출력하고 Stop한다:

```
---
👆 위 내용을 직접 실행해보세요.
실행이 끝나면 "완료" 또는 "다음"이라고 입력해주세요.
```

이 문구 이후에 어떤 도구 호출(AskUserQuestion 포함)이나 추가 텍스트도 출력하지 않는다.

### 블록 특수 규칙

- **Block 0 (Recap)**: 퀴즈 없음. Phase A에서 Day 2 회고 + `my-account-sync` 동작 점검 안내 → Stop. Phase B에서 동작 여부만 확인하고, 안 되면 5분 트러블슈팅으로 넘긴다.
- **Block 1 (Clarify)**: **체험 블록**. Phase A에서 참가자가 **일부러 모호하게** 시켜보게 하고("아웃리치 하나 써줘") → 결과 관찰 → 같은 요청을 이번엔 Claude가 **먼저 되묻게** 하는 두 단계를 안내 → Stop. Phase B에서 두 결과 비교 소감(정답 없음) + 핵심 메시지 퀴즈 1문항.
- **Block 2 (Pick)**: **선택 블록**. Phase A에서 템플릿 4종을 각 1줄로 소개 + "내 주간 루틴에서 제일 시간 먹는 것" 선택 기준 안내 → "준비되면 '선택'이라고 입력" → Stop. Phase B에서 **AskUserQuestion으로 택1**하고 고른 템플릿을 확정한 뒤 Block 3으로 넘어간다.
- **Block 3 (Customize)**: **메인 블록 · 인터랙티브**. Phase A에서 0.5→1 개념 + 슬롯 3곳 예고 + "Claude가 템플릿을 복사하고 CLAUDE.md를 읽어 슬롯 초안을 제안한다"고 안내 → "준비되면 '시작'" → Stop. Phase B에서 (1) 템플릿을 `.claude/skills/`로 복사 (2) CLAUDE.md를 Read해 슬롯 초안 제안 (3) 참가자와 다듬어 파일 저장. **이 블록만 Phase B에서 도구(Bash·Read·Write·AskUserQuestion)를 적극 사용한다.**
- **Block 4 (Run)**: Phase A에서 실제 딜 1개로 스킬 실행 안내 + "결과를 고치지 말고 관찰, 아쉬운 점 1개만 메모" → Stop. Phase B에서 결과 소감(정답 없음) + 아쉬운 점 1개를 메모로 남긴다("내일 검증에서 쓴다").
- **Block 5 (Wrap)**: Phase A에서 오늘 정리(Clarify + 0.5→1의 관계) → Stop. Phase B에서 자가진단 퀴즈 3문항 연속 → Day 4 예고로 마무리.

## Templates 인터페이스 (Block 2~3에서 사용)

Day 3 실습용 템플릿은 `templates/` 아래 4종이다. **Day 3 스킬은 이 인터페이스만 참조한다 — 템플릿 내용을 다시 쓰지 않는다.**

| 템플릿 | 입력 → 출력 | 이런 사람에게 |
|--------|------------|--------------|
| `call-prep` | 계정명 + 미팅 성격 → 콜 프렙 브리핑 | 미팅 준비에 매번 시간 쓰는 사람 |
| `battlecard` | 계정명 + 경쟁 MMP + 갱신 시기 → 이관 설득 카드 | 경쟁사 이관·갱신 딜이 많은 사람 |
| `meeting-followup` | 미팅 노트(Circleback) → 액션아이템 + CRM 제안 + 국/영문 팔로업 메일 초안 | 미팅 후 팔로업이 밀리는 사람 |
| `draft-outreach` | 프로스펙트 정보 → 아웃리치 초안 2버전 | 신규 아웃리치를 자주 쓰는 사람 |

- 각 템플릿은 `templates/<이름>` 에 있고, **파일 상단에 "수정 안내" 주석**과 **본문에 `{{슬롯}}` 3개**가 있다.
- **슬롯의 정확한 이름·위치는 하드코딩하지 않는다.** Block 3에서 해당 템플릿을 Read해 상단 주석을 보고 3개 슬롯을 그때 찾아낸다. (슬롯은 대체로 세그먼트/맥락 · 톤 · 금지어/용어 3축이다.)

## 진행 규칙

- 한 번에 한 블록씩 진행한다.
- "다음", "skip", 블록 번호/이름으로 이동한다.
- 참가자는 **터미널 초보 세일즈**다. 파일 복사·열기·수정은 **Claude가 대신 해준다** — 에디터(Cursor/VSCode)를 강요하지 않는다. "제가 복사하겠습니다", "제가 이 줄을 이렇게 바꿨습니다"처럼 Claude가 손을 대고 결과를 보여준다.
- 예시에 고객사·직원 **실명을 쓰지 않는다.** "글로벌 커머스 앱 A사"처럼 익명으로 안내한다.
- 오늘의 목적지는 **Block 3에서 동작하는 내 스킬 1개를 만드는 것**이다. 앞뒤 블록은 여기로 가기 위한 체험과 마무리다.

## 시작

스킬 시작 시 아래 안내와 함께 AskUserQuestion으로 어디서 시작할지 물어본다.

> 오늘은 **"잘 시키는 법"**을 배웁니다. 먼저 모호하게 시켜서 쓰레기가 나오는 걸 직접 보고(①), 그다음 운영진 템플릿 하나를 골라 내 딜 맥락으로 고쳐(②) **동작하는 내 스킬 1개**를 만들어 갑니다.

| Block | 주제 | 내용 |
|-------|------|------|
| 0 | Recap | Day 2 회고 — my-account-sync 잘 도는지 점검 |
| 1 | Clarify | 모호하게 vs 되묻게 — GIGO 체험 |
| 2 | Pick | 템플릿 4종 중 내 루틴에 맞는 하나 고르기 |
| 3 | Customize | 슬롯 3곳을 내 맥락으로 수정 (오늘 산출물) |
| 4 | Run | 실제 딜 1개로 실행 + 아쉬운 점 메모 |
| 5 | Wrap | 자가진단 + Day 4 예고 |

```json
AskUserQuestion({

    "question": "어디서부터 시작할까요?",

      {"label": "체험부터 (Block 1)", "description": "회고 건너뛰고 Clarify 체험부터"},
      {"label": "바로 만들기 (Block 3)", "description": "템플릿 골라뒀으면 커스터마이징부터"},
      {"label": "전체 훑기", "description": "각 블록 한 줄 요약만 먼저 보기"}

```
