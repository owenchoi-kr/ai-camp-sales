# Block 0: Recap — Day 2 회고 + my-account-sync 점검

> Day 2에서 만든 `my-account-sync` 스킬이 지금도 도는지 눈으로 확인하고 넘어간다. 퀴즈 없음.
> 이 블록의 목적: **오늘 커스터마이징의 출발선(어제 만든 스킬 + CLAUDE.md)이 살아있는지 점검.**

## EXPLAIN

어제 무엇을 했는지 한 화면으로 되짚는다.

```
Day 1  →  CLAUDE.md          내 담당 계정·톤·금지어 (내 기본값)
Day 2  →  my-account-sync    계정명 하나 넣으면 히스토리 브리핑이 나오는 내 스킬
                             (Gmail·Circleback·Notion·Slack 소스 연결)
Day 3  →  오늘               그 위에 "반복 작업 템플릿"을 얹어 내 것으로 만든다
```

오늘 하는 일은 **어제 만든 재료 위에 벽돌 하나를 더 얹는 것**이다. 그래서 시작 전에, 어제 만든 게 지금도 도는지부터 확인한다. 어제 잘 됐어도 세션을 새로 켜면 MCP 연결이 풀려 있을 수 있다.

## EXECUTE

**1. Day 2 스킬을 한 번 돌려본다.** 실제 담당 계정 하나(실명이 부담되면 "글로벌 커머스 앱 A사"처럼 익명)로:

```
/my-account-sync 글로벌 커머스 앱 A사
```

또는 자연어로: `A사 계정 히스토리 브리핑 뽑아줘`

**2. 결과가 나오면 OK.** 어제처럼 히스토리 브리핑이 나오면 준비 완료다. 오늘은 여기에 얹는다.

**3. 안 나오면 — 아래를 확인한다** (대부분 여기서 해결):

| 증상 | 확인 |
|------|------|
| MCP가 안 불린다 | `/mcp` 입력 → 목록에 Circleback·Notion·Slack이 떠 있나. 없으면 claude.ai 재로그인 후 Claude Code 재시작 |
| 스킬이 호출이 안 된다 | `/my-account-sync` 대신 자연어로 "계정 히스토리 브리핑"이라고 시도 |
| 결과가 비어 있다 | 계정명 철자 확인. 그래도 비면 소스에 그 계정 데이터가 아직 없는 것 — 다른 계정으로 시도 |

> 5분 안에 안 되면 손 들고 운영진 사이드 세션으로 넘어간다. 어제 스킬이 완벽하지 않아도 오늘 진도는 나갈 수 있다 — 오늘은 **다른 스킬**을 새로 만든다.
