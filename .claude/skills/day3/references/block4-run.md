# Block 4: Run — 실제 딜 1개로 실행 + 아쉬운 점 메모

> 방금 만든 스킬을 진짜 딜 하나에 돌려본다. **결과를 고치지 않는다.** 관찰하고, 아쉬운 점 1개를 메모로 남긴다.
> 이 블록의 목적: "돌아가는 걸 눈으로 확인" + "내일 검증에서 쓸 재료(아쉬운 점) 확보".

### 왜 지금 고치지 않고 관찰만 하나

첫 결과는 대체로 70%다. 그걸 지금 붙잡고 완벽하게 다듬으면 시간만 녹는다. 오늘 할 일은 **"이 스킬이 실제 딜에서 어떻게 나오나"를 눈으로 보는 것**과, **"어디가 아쉬운가"를 한 줄로 남기는 것**뿐이다.

그 아쉬운 점 메모는 버리는 게 아니다. **내일(Day 4) "까다로운 프로스펙트" 페르소나가 여러분 결과물을 채점**할 때, 바로 그 재료로 쓴다. 오늘 느낀 아쉬움이 내일 개선의 출발점이 된다.

```
오늘:  만든 스킬 → 실제 딜 1개로 실행 → 관찰(고치지 않음) → 아쉬운 점 1개 메모
내일:  그 메모 + 까다로운 프로스펙트 채점 → 개선
```

## EXECUTE

**1. 실제 딜 1개를 고른다.** 지금 진행 중인 딜/계정 하나. (실명이 부담되면 "글로벌 커머스 앱 A사"처럼 익명으로.)

**2. 방금 만든 스킬을 그 딜로 실행한다.** Block 3에서 만든 스킬 이름으로:

```
/<내가-만든-스킬-이름> 글로벌 커머스 앱 A사
```

또는 자연어로: `A사로 [콜 프렙 / 배틀카드 / 팔로업 / 아웃리치] 뽑아줘`

**3. 나온 결과를 그대로 읽는다. 고치지 않는다.** 관찰 포인트:

- 슬롯에 넣은 **내 톤·세그먼트**가 결과에 실제로 묻어나는가?
- **금지어**는 잘 피했는가?
- 어디가 제일 아쉬운가? (딱 1곳만)

**4. 아쉬운 점 1개를 한 줄로 정한다.** "여기가 제일 아쉽다" 하나만. 여러 개 말고 하나.

## QUIZ

> Block 4는 정답 있는 퀴즈가 아니라, 결과 소감 + 아쉬운 점을 메모로 남기는 블록이다.

먼저 한 줄 정리:

```
방금 한 것:
- 내가 만든 스킬이 실제 딜에서 돌아가는 걸 확인했다
- 70%면 정상. 나머지 30%가 내일의 재료다
```
