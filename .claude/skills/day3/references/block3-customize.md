# Block 3: Customize — 슬롯 3곳을 내 맥락으로 (오늘의 산출물)

> 오늘 하루의 목적지. 고른 템플릿을 `.claude/skills/`로 복사하고, `{{슬롯}}` 3곳을 내 맥락으로 바꿔 **동작하는 내 스킬 1개**를 만든다.
> 이 블록만 STOP PROTOCOL을 조금 다르게 쓴다: Phase A는 설명 + 예고 후 Stop, Phase B에서 Claude가 도구(Bash·Read·Write·AskUserQuestion)를 적극 써서 인터랙티브하게 진행한다.
> 참가자는 터미널 초보다. **복사도 수정도 Claude가 대신 한다** — 에디터를 열게 하지 않는다.

## EXPLAIN (Phase A)

> **Phase A 시작 시 반드시 아래 형태로 출력한다:**
> ```
> 📖 공식 문서: https://code.claude.com/docs/ko/skills
> ```

한 문단으로 개념을 짚는다:

> 운영진 템플릿은 이미 **70%가 채워진 0.5판**이다. 콜 프렙이든 아웃리치든, 뼈대·순서·되묻기(Clarify)는 다 들어 있다. 비어 있는 건 딱 하나 — **"이게 누구의 스킬인가"**다. 그 빈칸이 `{{슬롯}}`이다. 슬롯 3곳만 내 맥락으로 채우면, 남의 템플릿이 내 스킬이 된다.

이제 채울 **슬롯 3곳**을 예고한다 (템플릿마다 이름은 조금씩 다르지만 대체로 이 3축이다):

1. **세그먼트 / 맥락** — 내가 담당하는 계정군·버티컬. 템플릿이 "일반 세일즈" 기본값 대신 내 고객을 가정하게 한다
2. **톤** — 고객 앞 언어 스타일 (짧고 담백 / 관계 중심 / 데이터 설득)
3. **금지어 / 용어** — 절대 쓰면 안 되는 표현 + 우리 팀이 쓰는 용어

그리고 이 블록이 어떻게 진행되는지 알려준다:

> 제가 **① 고른 템플릿을 여러분 스킬 폴더로 복사**하고, **② Day 1에 만든 CLAUDE.md를 읽어** 슬롯 3곳의 초안을 먼저 제안하겠습니다. 여러분은 그 초안을 보고 다듬기만 하면 됩니다. 파일은 제가 저장합니다.

Phase A 종료 문구:

```
---
👆 준비되면 "시작"이라고 입력해주세요.
제가 템플릿을 복사하고, CLAUDE.md를 읽어 슬롯 초안을 제안하겠습니다.
```

여기서 Stop한다. (Phase A에서는 Bash·Read·Write·AskUserQuestion을 호출하지 않는다.)

## EXECUTE (Phase B) — 커스터마이징 진행 규칙

사용자가 "시작"이라고 하면, 아래 순서로 진행한다. **참가자에게 파일 편집을 시키지 않는다 — 모든 파일 작업은 Claude가 한다.**

### 1단계 — 템플릿을 내 스킬 폴더로 복사

Block 2에서 고른 템플릿 이름을 `<이름>`이라 하자 (call-prep / battlecard / meeting-followup / draft-outreach 중 하나).

먼저 템플릿 구조를 확인하고 복사한다:

```bash
ls templates/<이름>
cp -r templates/<이름> .claude/skills/<이름>
```

- `templates/<이름>`가 폴더가 아니라 단일 파일이면 그에 맞게 복사한다(`ls` 결과로 판단).
- 복사 후 "복사 완료: `.claude/skills/<이름>` 에 여러분 스킬이 생겼습니다"라고 알린다.
- 같은 이름이 `.claude/skills/`에 이미 있으면 덮어쓰지 말고, "이미 있는데 새로 덮어쓸까요, 다른 이름으로 만들까요?"라고 묻는다.

### 2단계 — 슬롯 3곳을 찾는다

복사한 스킬 파일을 Read한다 (예: `.claude/skills/<이름>/SKILL.md`). **파일 상단의 "수정 안내" 주석**을 읽어 `{{슬롯}}` 3개가 각각 무엇인지 파악한다. 슬롯 이름을 하드코딩하지 말고, **그 파일에 실제로 있는 슬롯**을 그대로 쓴다.

파악한 3개 슬롯을 참가자에게 한 줄씩 보여준다: "이 템플릿의 채울 곳은 이 3곳입니다 — ①… ②… ③…".

### 3단계 — CLAUDE.md를 읽어 슬롯 초안을 제안

Day 1에 만든 `CLAUDE.md`(프로젝트 루트)를 Read한다. 거기 담긴 **담당 세그먼트·톤·금지어**를 재료로 각 슬롯의 초안을 만든다.

- CLAUDE.md가 없으면: "Day 1 프로필이 없네요. 세 가지만 짧게 물어볼게요"라고 하고 AskUserQuestion으로 세그먼트·톤·금지어를 빠르게 받는다.
- 초안은 완성본이 아니라 **제안**이다. "제가 이렇게 채워봤습니다. 맞나요?"로 보여준다.

슬롯 초안 제안 예시(톤 슬롯):

```
슬롯 ②(톤) 초안 — CLAUDE.md에서 가져옴:
  "임원 회람 가능성 고려. 결론 먼저, 한 문단 3줄 이내. 과장 없이 담백하게."
이대로 둘까요, 고칠까요?
```

세 슬롯을 한 번에 쏟지 말고, **한 슬롯씩** 제안 → 참가자 확인 → 다음 슬롯으로 넘어간다. 선택지가 명확한 건 AskUserQuestion, 서술이 필요한 건 자유서술로 받는다. 참가자가 "그대로 좋아요"라고 하면 초안을 그대로 확정한다 — 0.5→1이 목표지 완벽이 아니다.

### 4단계 — 파일에 저장 (반드시 실제로 저장한다)

세 슬롯이 확정되면:

1. Edit 도구로 복사본 파일의 `{{슬롯}}` 3곳을 확정한 내용으로 **실제 치환**한다. (참가자가 직접 편집하지 않는다.)
2. 치환 후 바뀐 부분을 참가자에게 보여주며 "이렇게 채웠습니다"라고 확인한다.
3. "이제 `/<이름>`으로 이 스킬을 부를 수 있습니다. 다음 블록에서 실제 딜로 돌려봅니다"라고 안내한다.

> 세 슬롯 말고 다른 곳은 건드리지 않는다. **2곳은 그대로 두고 필요한 곳만** 손대는 게 0.5→1의 핵심이다.

### 막히면 (Troubleshooting)

| 증상 | 대응 |
|------|------|
| `{{슬롯}}`을 못 찾겠다 | 파일 상단 "수정 안내" 주석을 다시 Read. 주석에 슬롯 위치가 적혀 있다 |
| CLAUDE.md가 얇다 | AskUserQuestion으로 세그먼트·톤·금지어 3개만 빠르게 받아 채운다 |
| 복사가 안 된다 | `.claude/skills/` 폴더가 있는지 `ls .claude/skills` 로 확인 후 재시도 |

## QUIZ

    "question": "방금 만든 스킬, 지금 상태 어떤가요?",
    "header": "산출물 확인",
    "options": [

      {"label": "1~2곳만 채웠다", "description": "괜찮다. 나머지는 실행해보고 채워도 된다 — Block 4로"},
      {"label": "저장이 안 됐다", "description": "파일 치환/저장부터 다시 확인"}
    ],
    "multiSelect": false
  }]
})
```
