# Block 3: Graduate — 내 것으로 가져가기 (전역 승격)

> 지금까지 만든 건 이 캠프 폴더 안에서만 돈다. 2주차엔 내 실제 업무에 써야 한다.
> 이 블록의 메시지: **"캠프 폴더는 연습장. 검증 끝난 것만 골라서 실제 환경으로 승격한다."**

## EXPLAIN (Phase A)

> **Phase A 시작 시 반드시 아래 형태로 출력한다:**
> ```
> 📖 공식 문서: https://code.claude.com/docs/ko/skills
> ```

### 지금까지 만든 건 이 폴더 안에서만 돈다

Day 1~3에 만든 `CLAUDE.md`와 스킬(`my-account-sync`, 어제 커스터마이징한 템플릿 스킬)은 지금 **이 캠프 폴더(`ai-camp-sales`) 안에만** 있다. 다른 폴더를 열고 claude를 켜면 그것들은 없다. `/call-prep`을 쳐도 "그런 스킬 없다"고 나온다. 프로젝트 폴더 안의 `.claude/`는 그 폴더에서만 작동하기 때문이다.

### 2주차엔 내 실제 업무 폴더에서 해야 한다

2주차 프로젝트는 실습이 아니라 **내 진짜 딜, 내 진짜 파이프라인**에 적용하는 것이다. 그건 이 연습용 캠프 폴더가 아니라 **내가 실제로 일하는 폴더**에서 하게 된다. 그러려면 지금 만든 것들을 **전역(`~/.claude/`)으로 옮겨야** 한다. 전역에 두면 어느 폴더에서 claude를 켜든 `/call-prep`, "내 계정 프로필 알아?"가 그대로 작동한다.

### 캠프 폴더 = 연습장, 전역 = 실제 환경

여기서 중요한 건 **아무거나 다 전역에 깔지 않는다**는 것이다. 바로 앞 Block 1에서 우리는 까다로운 프로스펙트에게 결과물을 채점받았다. 그 검증을 통과한 것 — **믿을 수 있게 다듬은 것만** 골라서 승격한다. 연습장에서 이것저것 만들어보고, 실전에 데려갈 건 검증 끝난 것만 고르는 것이다. 이게 Day 4 내내 관통하는 원리(만든 걸 그냥 믿지 않는다)와 같은 결이다.

## EXECUTE (Phase B) — 전역으로 복사 + CLAUDE.md 안전 병합

참가자가 돌아오면, 아래 순서로 **Claude가 대신** 처리한다. 참가자는 터미널 초보라는 전제로, 무엇을 하는지 한 줄씩 알려주며 진행한다.

### 1. 가져갈 스킬 고르기 (AskUserQuestion)

```json
AskUserQuestion({
  "questions": [{
    "question": "어떤 스킬을 전역(~/.claude/)으로 가져갈까요? (2주차에 어디서든 쓰게 됩니다)",
    "header": "가져갈 스킬",
    "options": [
      {"label": "my-account-sync", "description": "계정명 하나로 히스토리 브리핑 (Day 2)"},
      {"label": "내 템플릿 스킬", "description": "call-prep 등 어제 만든 것 (Day 3)"},
      {"label": "둘 다", "description": "검증 통과한 것 위주로 둘 다 가져간다"}
    ],
    "multiSelect": true
  }]
})
```

> 참가자가 만든 스킬 폴더 이름이 확실치 않으면 먼저 `.claude/skills/` 안을 확인(`ls`)해 실제 폴더명을 짚어준다.

### 2. 전역 스킬 폴더로 복사

전역 스킬 폴더가 없을 수 있으니 먼저 만든다. 그다음 고른 스킬을 복사한다.

```bash
mkdir -p ~/.claude/skills
cp -r .claude/skills/<스킬이름> ~/.claude/skills/<스킬이름>
```

- `<스킬이름>`을 실제 폴더명으로 바꿔 각 스킬마다 한 번씩 실행한다.
- 복사 후 `ls ~/.claude/skills/`로 들어갔는지 보여준다.

### 3. CLAUDE.md 병합 — ⚠️ **덮어쓰기 절대 금지 (가장 중요)**

**전역 CLAUDE.md(`~/.claude/CLAUDE.md`)를 절대 덮어쓰지 않는다.** 참가자가 이미 개인 설정을 가지고 있을 수 있고, 비개발자가 그걸 실수로 날리면 복구가 어렵다. 반드시 아래 분기를 따른다:

- **`~/.claude/CLAUDE.md`가 이미 있으면:** 먼저 물어본다 —
  > "기존 개인 설정(`~/.claude/CLAUDE.md`)이 있네요. 덮어쓰지 않고, 캠프에서 만든 프로필을 **그 아래에 이어붙일까요?**"
  
  참가자가 동의하면 **append만** 한다. 캠프 CLAUDE.md 내용을 기존 파일 끝에 구분선과 함께 덧붙인다. 기존 내용은 한 글자도 지우지 않는다.
  ```bash
  printf '\n\n---\n# (캠프에서 가져온 프로필)\n\n' >> ~/.claude/CLAUDE.md
  cat CLAUDE.md >> ~/.claude/CLAUDE.md
  ```
- **`~/.claude/CLAUDE.md`가 없으면:** 그때만 캠프 폴더 CLAUDE.md를 그대로 복사한다.
  ```bash
  cp CLAUDE.md ~/.claude/CLAUDE.md
  ```

> 판단이 서지 않으면 항상 "있다"고 가정하고 물어본 뒤 append한다. 덮어쓰기는 어떤 경우에도 하지 않는다.

### 4. 전역 작동 확인

옮긴 게 진짜 어디서든 도는지 확인시킨다:

- **다른 폴더(예: 홈)에서** claude를 새로 켜고 `/call-prep`(또는 옮긴 스킬 이름)을 쳐본다.
- 또는 "내 계정 프로필 알아?"라고 물어 CLAUDE.md가 읽히는지 본다.
- 캠프 폴더가 아닌 곳에서 작동하면 승격 성공이다.

```
✅ 이제 어느 폴더에서든 이 스킬들이 돕니다.
2주차엔 캠프 폴더가 아니라 '내 실제 업무 폴더'에서 진행하세요.
```

    },
    {
