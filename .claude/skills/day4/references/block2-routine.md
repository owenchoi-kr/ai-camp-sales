# Block 2: Routine — 내 월요일 아침 30분 루틴 조립

> 지금까지 만든 스킬은 흩어져 있다. 이걸 **매주 같은 시간에 돌아가는 순서**로 엮는다.
> 이 블록의 메시지: **"스킬 하나하나가 아니라, 스킬을 잇는 '루틴'이 시간을 아낀다."**

## EXPLAIN (Phase A)

> **Phase A 시작 시 반드시 아래 형태로 출력한다:**
> ```
> 📖 공식 문서: https://code.claude.com/docs/ko/skills
> ```

### 조립이란 — 스킬을 순서로 엮기

Day 2~3에 만든 건 개별 부품이다: `my-account-sync`(계정 히스토리), 어제 커스터마이징한 스킬(call-prep / meeting-followup 등). 부품은 필요할 때 하나씩 꺼내 쓴다. 하지만 **매주 반복되는 일은 순서가 정해져 있다.** 그 순서를 한 번 적어두면, 월요일 아침마다 고민 없이 그대로 돈다.

### 예시 — 월요일 아침 30분 루틴

```
월요일 09:00 (30분)
  1. my-account-sync    →  이번 주 미팅 잡힌 계정들의 히스토리를 한 번에 뽑는다
  2. call-prep          →  그중 오늘/내일 미팅 계정의 콜 프렙 브리핑을 만든다
  3. (미팅이 끝나면)
     meeting-followup   →  미팅 직후 액션아이템·팔로업 메일 초안을 뽑는다
```

핵심은 **트리거(언제) + 순서(무엇을 어떤 차례로)**다. "월요일 아침"이라는 트리거가 있으면, 매주 백지에서 시작하지 않는다.

> 어제 만든 스킬이 call-prep이 아니라 battlecard나 draft-outreach여도 된다. 그 스킬을 넣어 **내 실제 주간 흐름**에 맞게 순서를 짠다. 정답 루틴은 없다.

### 검증을 루틴에 넣기 (Block 1 연결)

여유가 되면 아웃리치·중요 메일은 보내기 전에 **까다로운 프로스펙트 채점(Block 1)**을 한 스텝 끼운다. "쓰기 → 채점 → 1곳 고치기 → 보내기". 이게 결과물을 믿을 수 있게 만드는 습관이다.

## EXECUTE (Phase A에서 안내만)

참가자에게 안내한다:

**내 월요일 아침(또는 내 주간 시작) 루틴을 딱 3줄로 적어보세요.** 위 예시를 참고하되, 내 실제 스킬과 흐름으로.

```
내 주간 루틴 (트리거: ____요일 ____시)
1. (어떤 스킬 → 무엇을)
2. (어떤 스킬 → 무엇을)
3. (어떤 스킬 → 무엇을)
```

> 3줄이면 충분하다. 스킬이 2개뿐이면 2줄이어도 된다. 완벽한 루틴이 아니라 **매주 반복하는 실제 순서**를 적는 게 목적이다.

Phase A 종료 문구:

```
---
👆 내 루틴을 3줄로 적어서 알려주세요.
적어주시면 CLAUDE.md에 "내 주간 루틴" 섹션으로 넣어드립니다.
```

여기서 Stop한다. (Phase A에서는 Edit·AskUserQuestion을 호출하지 않는다.)

## EXECUTE (Phase B) — CLAUDE.md에 루틴 추가

참가자가 루틴 3줄을 주면:

1. 트리거(요일·시각)가 빠졌으면 한 번만 묻는다: "이 루틴, 주로 언제 도나요? (예: 월요일 오전)"
2. 흐름이 어색하면 한 번만 제안한다(예: "팔로업은 미팅 뒤라 순서상 맨 뒤가 맞겠네요"). 취조하지 않는다 — 참가자 루틴을 존중한다.
3. **프로젝트 루트 `CLAUDE.md`에 아래 섹션을 추가(Edit)한다.** 기존 CLAUDE.md 끝에 덧붙인다. CLAUDE.md가 없으면 Write로 새로 만들되, 이 섹션만 넣는다.

추가할 섹션 형식:

```markdown
## 내 주간 루틴
- 트리거: {요일 + 시각}
1. {스킬} → {무엇을}
2. {스킬} → {무엇을}
3. {스킬} → {무엇을}
- (선택) 중요 아웃리치는 보내기 전 '까다로운 프로스펙트' 채점 → 1곳 수정
```

4. 추가한 내용을 그대로 보여주고 안내한다: "이제 '내 주간 루틴 돌려줘' 한 마디면 이 순서대로 진행할 수 있어요."

```

> 정리 한 줄: **부품(스킬)이 아니라 순서(루틴)가 매주 시간을 아낀다.** 다음 블록에선 이 루틴의 빈 스텝을 채울 2주차 프로젝트를 정한다.
