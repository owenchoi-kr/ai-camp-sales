# Block 5: Wrap — 1주차 수료 + 자가진단 + 2주차 운영 안내

> 1주차를 닫는다. 4일 산출물을 한 화면으로 확인하고, 자가진단 3문항으로 점검하고, 2주차 운영을 안내한다.

## EXPLAIN (Phase A)

4일간 만든 걸 한 화면으로 되짚는다.

**4일 산출물 체크리스트:**

```
□ CLAUDE.md              (Day 1)  내 담당 계정·톤·금지어 + 주간 루틴(Day 4에 추가)
□ my-account-sync 스킬    (Day 2)  계정명 하나면 히스토리 브리핑 (4소스 연결)
□ 내 템플릿 스킬 1개       (Day 3)  call-prep / battlecard / meeting-followup / draft-outreach 중 1개
□ my-project.md          (Day 4)  2주차 프로젝트 스코핑 1장  ← 오늘 만든 것
```

**4일의 흐름 한 줄:** 프로필(Day 1) → 도구로 맥락 채우기(Day 2) → 내 맥락으로 커스터마이징(Day 3) → 검증·조립·스코핑(Day 4). 만들고(1~3일), 믿을 수 있게 만들고 다음을 정했다(4일).

**오늘 얻은 습관 2개:**
- **검증** — 만든 걸 다른 눈(까다로운 프로스펙트)으로 채점한다. 만든 사람은 공정히 못 본다.
- **조립** — 흩어진 스킬을 매주 돌아가는 루틴으로 엮는다.

Phase A 종료 문구:

```
---
👆 4일 산출물을 확인했으면 "다음"이라고 입력해주세요.
자가진단 퀴즈 3개와 2주차 운영 안내를 드립니다.
```

여기서 Stop한다.

## QUIZ (Phase B) — 자가진단 3문항 연속

사용자가 돌아오면 3문항을 **연속으로** 출제한다. 시험이 아니라, 1주차에 만든 게 살아있는지 점검하는 자가진단이다. 각 문항에 짧은 피드백을 준다.

### 자가진단 1

```json
AskUserQuestion({
  "questions": [{
    "question": "왜 내가 만든 결과물을 '나 말고 다른 관점'에게 채점시켰을까요?",
    "header": "자가진단 1/3",
    "options": [
      {"label": "만든 사람은 자기 맥락 때문에 빈 곳을 못 봐서", "description": "정답. Generator≠Evaluator"},
      {"label": "AI가 사람보다 정확해서", "description": "정확함이 아니라 '관점 분리'가 핵심"}
    ],
    "multiSelect": false
  }]
})
```

> 피드백: 핵심은 정확도가 아니라 관점이다. 받는 사람은 내 맥락을 모른 채 읽는다 — 그 상태를 재현한 게 까다로운 프로스펙트다.

### 자가진단 2

```json
AskUserQuestion({
  "questions": [{
    "question": "내 CLAUDE.md에 '주간 루틴'이 트리거(언제)와 함께 적혀 있나요?",
    "header": "자가진단 2/3",
    "options": [
      {"label": "트리거 + 순서가 있다", "description": "좋다 — 월요일마다 백지에서 시작 안 한다"},
      {"label": "순서만 있고 트리거가 없다", "description": "'언제'를 붙이면 습관이 된다. 지금 한 줄 추가"}
    ],
    "multiSelect": false
  }]
})
```

> 피드백: "월요일 오전"처럼 트리거가 있으면 루틴이 실제로 돈다. 순서만 있으면 매번 "언제 하지?"에서 미뤄진다.

### 자가진단 3

```json
AskUserQuestion({
  "questions": [{
    "question": "내 my-project.md, 2주 안에 끝낼 수 있게 범위가 좁나요?",
    "header": "자가진단 3/3",
    "options": [
      {"label": "좁다 — 실제 딜 1개에 적용", "description": "합격. 좁게 잡고 하나에 적용하는 게 정답"},
      {"label": "아직 넓다", "description": "'범위 밖'을 더 채워라. 넓으면 못 끝낸다"}
    ],
    "multiSelect": false
  }]
})
```

> 피드백: 2주는 짧다. 자동 발송·팀 일반화는 대체로 범위 밖. 실제 딜 1개에 적용해 before/after 1장을 만드는 게 목표다.

### 마무리 — 2주차 운영 안내

3문항이 끝나면 아래를 안내하고 1주차를 닫는다:

```
🎓 1주차 수료 — 축하합니다.

4일 만에 만든 것:
- CLAUDE.md (프로필 + 주간 루틴)
- my-account-sync 스킬
- 내 템플릿 스킬 1개
- my-project.md (2주차 스코핑)

📅 2주차 운영
- 이번 주: my-project.md의 프로젝트를 내 실제 딜 1개에 적용해 만든다.
  → 스킬은 이미 전역으로 옮겼으니(Day 4 가져가기), 캠프 폴더가 아니라 '내 실제 업무 폴더'에서 진행한다.
- 수요일: Slack 스레드로 중간 체크인 1회 (진행 상황 + 막힌 곳).
  → 막히면 그날 물어보기. 금요일에 몰아서 오면 못 도와줍니다.
- 금요일: 쇼케이스 — before/after 1장 (걸리던 시간 → 아낀 시간), 3분 발표. 데모 라이브면 가산점.

한 가지만 기억하세요: 새로 대단한 걸 만들지 마세요.
매주 반복되던 시간 낭비 하나를, 실제 딜 하나에서 없애면 됩니다.

1주 동안 수고 많으셨습니다. 2주차 쇼케이스에서 만나요.
```

> 산출물이 일부 비어도 괜찮다고 덧붙인다 — 2주차에 채울 수 있다. 중요한 건 my-project.md 하나를 실제 딜에 적용해보는 것.
