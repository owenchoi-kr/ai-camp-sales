# Block 0: Recap — Day 1 회고 + 숙제 점검 + 오늘 예고

> Day 1을 되짚고, 어젯밤 숙제(도구 1개 연결)를 점검하고, 오늘 만들 것을 예고한다.
> 퀴즈 없음. Phase A는 회고+예고 → Stop. Phase B는 숙제 점검(AskUserQuestion)만.

## EXPLAIN (Phase A)

### 어제 만든 것 — CLAUDE.md

Day 1에 여러분은 **CLAUDE.md** 한 장을 만들었다. 내 담당 계정·세그먼트·톤·금지어를 담은 "내 기본값". 이제 이 파일이 모든 대화의 출발점이다.

그리고 핵심 기능 4개를 만났다:

```
CLAUDE.md (Memory) ── 매 세션 자동으로 읽히는 "내 기본값"   ← 어제 만든 것
Skill ────────────── 반복 업무를 한 줄로                    ← 오늘 만든다
MCP ──────────────── 외부 도구(Gmail·Circleback·Notion·Slack) 연결  ← 오늘의 재료
Subagent ────────── 여러 소스를 동시에 수집                 ← 오늘의 엔진
```

### 오늘 예고 — 퇴근할 때 스킬이 하나 생긴다

Day 1 Block 1에서 봤던 콜 프렙 데모, 기억나는가? 그때는 미팅 하나로 브리핑을 만들었다.
오늘은 그걸 **여러분의 스킬**로 만든다.

> **`my-account-sync`** — 계정명 하나만 넣으면, 그 계정과 우리가 지금까지 주고받은
> 메일·미팅·문서·내부 논의·딜 현황을 싹 모아 **콜 프렙용 히스토리 브리핑 한 장**을 뽑아주는 스킬.

만드는 방식이 핵심이다. 빈 뼈대(STUB 5개)를 먼저 만들고, 소스를 **하나씩** 채운다.
채울 때마다 스킬을 돌려보며 점점 좋아지는 걸 눈으로 본다. 도구 연결(MCP)은 이 과정에서 자연스럽게 붙는다 — 따로 외울 필요 없다.

```
지금                                오늘 퇴근할 때
──────────                          ──────────────
콜 잡히면 →                          "이 계정 싱크해줘" (한 마디)
  Gmail 뒤지고                              │
  미팅노트 찾고                             │
  Notion 열고          ──▶            Claude가 5소스에서 알아서 수집
  Slack 검색                                │
  SFDC 딜 확인                        히스토리 브리핑 한 장!
= 30분+                              = 1~2분
```

Phase A 종료 문구:

```
---
👆 오늘 만들 것을 확인했으면 "다음"이라고 입력해주세요.
어젯밤 숙제(도구 1개 연결)부터 점검합니다.
```

여기서 Stop한다.

## QUIZ (Phase B) — 숙제 점검 (정답 없음)

사용자가 돌아오면 어젯밤 숙제를 점검한다. 맞고 틀리는 시험이 아니라, 오늘 실습 준비 상태를 파악하는 것이다.

```json
AskUserQuestion({
  "questions": [{
    "question": "어젯밤 숙제 — Gmail·Circleback·Notion·Slack 중 하나라도 연결해 오셨나요?\n(Claude Code에서 /mcp 입력 → 목록에 뜨면 연결된 것)",
    "header": "숙제 점검",
    "options": [
      {"label": "네, 1개 이상 연결했어요", "description": "좋아요 — 오늘 그 도구부터 바로 씁니다"},
      {"label": "gws(Gmail)는 되는 것 같아요", "description": "사내 CLI. Block 3에서 바로 확인합니다"},
      {"label": "아직 못 했어요", "description": "괜찮아요 — 오늘 블록마다 같이 연결합니다"},
      {"label": "연결이 맞는지 모르겠어요", "description": "Block 2~6에서 하나씩 확인하며 갑니다"}
    ],
    "multiSelect": false
  }]
})
```

> 피드백 방향:
> - **연결함**: "그 도구가 오늘 첫 소스가 됩니다. 나머지는 블록마다 같이 붙여요."
> - **못 함/모름**: "전혀 문제 없어요. 오늘은 블록마다 도구 하나씩 연결하는 시간이 있고, 안 되면 **붙여넣기 모드(Plan B)**로 똑같이 따라올 수 있어요. 연결이 목적이 아니라 스킬이 동작하는 걸 체험하는 게 목적입니다."

점검 후 Block 1로 넘어간다.
