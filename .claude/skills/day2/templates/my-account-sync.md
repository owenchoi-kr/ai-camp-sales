# My Account Sync

계정명 하나를 입력하면, 그 계정과 우리가 지금까지 주고받은 접점을 5개 소스에서 모아
**콜 프렙용 히스토리 브리핑 한 장**으로 정리한다.

## 입력

- **계정명**: 회사명 또는 이메일 도메인 (예: `회사명` 또는 `@company.com`)
- 계정명이 없으면 수집을 시작하기 전에 사용자에게 먼저 묻는다.

## 실행 흐름

<!-- 🔧 Block 7에서 완성 -->

이 스킬이 트리거되면 아래 순서로 실행한다.

### 1단계: 병렬 수집

계정명을 확인한 뒤, 5개 소스를 **동시에** 수집한다. 서로 의존성이 없으므로 병렬 실행이 가능하다.

```
[계정명] 수집 시작
  ├── [소스 1] Gmail 스레드 검색             ─┐
  ├── [소스 2] Circleback 미팅노트 검색       │
  ├── [소스 3] Notion 계정 문서 검색          ├── 병렬 실행
  ├── [소스 4] Slack 내부 논의 검색           │
  └── [소스 5] Salesforce Opportunity 검색   ─┘
수집 완료
```

각 소스 수집은 subagent(Agent 도구)로 실행한다:

```
[STUB - Block 7에서 소스별 Agent 호출 구체화]
```

### 2단계: 결과 통합

수집된 정보를 하나의 히스토리 브리핑으로 합친다.

통합 규칙:
- 시간순으로 최근 접점을 먼저
- 소스가 달라도 같은 사건은 하나로 묶는다 (메일 + 미팅이 같은 건이면 한 줄)
- 약속·다음 스텝·미해결 질문은 반드시 "다음 액션"에 모은다
- 수집 실패한 소스는 "수집 실패" 표시와 함께 사유 기록

### 3단계: 브리핑 저장 + 출력

```
저장 위치: accounts/{계정명}/YYYY-MM-DD-briefing.md
```

## 출력 포맷 — 히스토리 브리핑

<!-- 🔧 Block 7에서 완성 -->

```
[STUB - Block 7에서 출력 형식 선택 후 작성]

기본 골격:

# {계정명} 히스토리 브리핑 ({날짜})

## 한 줄 요약
(이 계정과 우리의 현재 관계를 한 줄로)

## 최근 접점 타임라인
- [날짜] [소스] 무슨 일이 있었나

## 지난 미팅에서 나온 것
- 논의 핵심 / 약속한 액션 아이템

## 열려 있는 것 · 다음 액션
- [ ] 답장 필요한 메일
- [ ] 미해결 질문 / 이행 안 된 약속

## 콜 들어가기 전 체크
- 지난번 약속이 지켜졌나
- 이번 콜의 목표 한 줄
```

## 커스터마이징 가이드

### 소스 추가하기

새 소스를 추가하려면 "소스 정의"에 같은 형식으로 추가한다:

```markdown
### 소스 6: {도구명}

| 항목 | 값 |
|------|-----|
| 연결 방법 | `{연결 방법}` |
| 수집 범위 | {범위} |

수집 방법:
{MCP 호출 방법 또는 CLI 실행 방법}

추출할 정보:
- {항목 1}
- {항목 2}
```

### 소스 제거하기

사용하지 않는 소스는 해당 "소스 N" 섹션 전체를 삭제하고,
실행 흐름의 병렬 수집 부분에서도 해당 줄을 제거한다.

### 수집 범위·정보 바꾸기

담당 세그먼트에 맞게 "수집 범위"와 "추출할 정보"를 고친다.
예: 엔터프라이즈는 미팅노트 비중을 높이고, PLG 계정은 Slack 지원 이력 비중을 높인다.
