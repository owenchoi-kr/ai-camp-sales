## 공통 리듬 — 소스 블록의 4스텝 안무 (Block 3~7 반복)

강사가 매 소스 블록에서 반복하는 흐름이다. 이게 점진 빌드의 엔진이다.

```
① AskUserQuestion으로 이 소스의 연결 경로 확정 (MCP됨 / 연결필요 / 붙여넣기)
② 선택 경로로 연결·테스트 (/mcp 확인 → 검색 함수 1회 테스트)
③ my-account-sync/SKILL.md의 해당 소스 STUB 줄을 실제 수집방법으로 교체
④ 같은 담당 계정으로 "{계정명} 계정 싱크해줘" 재실행 → 브리핑이 지난 블록보다 두꺼워진 것 확인
```

**강사 반복 대사**: "5개를 한 번에 붙이지 않습니다. 하나 채우고 바로 돌려서, 스킬이 눈앞에서 좋아지는 걸 봅니다." Block 3(메일 1소스 밋밋한 브리핑) → Block 7(같은 계정이 딜까지 붙은 완성 브리핑)의 **Before/After 대비**가 하루의 클라이맥스다.

공통 STOP PROTOCOL: 각 블록 2턴. Phase A 종료 문구 "👆 위 내용을 직접 실행해보세요. 끝나면 '완료' 또는 '다음'." **gws/MCP 연결 명령을 지어내지 않는다** — reference에 "운영진 배포 가이드 참조"면 그대로.

## Block 1 — MCP (6분)

- **🎤 강사**: `📖 .../mcp` + `📖 modelcontextprotocol.io`. MCP = AI의 USB-C = 표준 연결 규격. **연결 방법이 여러 가지**임만 짚기(CLI=gws, 커넥터=Slack·Gmail, MCP서버=Circleback·Notion, 붙여넣기=Salesforce). "지금 외울 필요 없다, 주인공은 스킬." **깊게 파지 않는다.**
- **⌨️ 참가자**: `/mcp` 입력해 현재 연결 상태 확인 (비어 있어도 OK)
- **✋ STOP**: `/mcp` 실행 안내 후 멈춤
- **✅ 체크**: "MCP를 한 마디로?" → 정답: 표준 연결 규격
- **⚠️ 막히는 곳**: 여기서 MCP를 깊게 설명하려는 유혹 → 참지 말 것. `/mcp` 목록 비어도 정상.

## Block 3 — Gmail 소스① (13분 · 첫 연결)

- **🎤 강사**: `📖 .../mcp` + `📖 gws CLI: 사내 배포 가이드`. "콜 전에 제일 먼저 뒤지는 게 메일함." 기본 = 사내 **gws CLI**(`gws gmail users messages list --params '{...q: from/to 도메인...}'`), 본문은 `messages get`. **3단 안전망**(gws / Gmail 커넥터 / 붙여넣기).
- **⌨️ 참가자**: 경로 선택 → 연결·테스트 → 소스1 STUB 교체 → **실제 계정 하나로 "{계정명} 계정 싱크해줘" 첫 중간 실행 → 메일만으로 미니 브리핑**
- **✋ STOP**: 소스 1개 미니 브리핑 + 진행판(소스1 ✅ / 2~5 STUB) 표시 후 멈춤
- **✅ 체크**: "gws 없거나 정책으로 막히면 실습 못 하나?" → 정답: 붙여넣기로 가능
- **⚠️ 막히는 곳 (핵심)**: gws 명령 실패 → Claude가 지어내지 않음, `gws schema gmail.users.messages.list`로 q문법 확인, 안 되면 **Plan B 커넥터**. 회사 정책상 커넥터 차단 → gws나 붙여넣기. 둘 다 실패 → 붙여넣기(스레드 복사). **보안**: 고객 실명 금지, 도메인·이니셜로.
- **🖥 슬라이드**: slide-02 지도 카드 ③ (Block 3~7 공통)

## Block 7 — Salesforce 소스⑤ (8분 · 딜 데이터)

- **🎤 강사**: `📖 SFDC 웹: 계정 페이지에서 Opportunity·활동 복사(붙여넣기 = 기본)` + `📖 Salesforce 연결(sf CLI/MCP): 운영진 배포 가이드`. "메일·미팅·문서·내부논의까지 왔다. 마지막 한 조각 — **딜 그 자체.**" Salesforce = **딜의 시스템 오브 레코드**(Stage·Amount·Close Date·Owner가 정형 데이터로 박힌 곳). 콜 프렙에서 "이 계정, 파이프라인상 어디쯤이고 얼마짜리인가"를 주는 유일한 소스.
- **⌨️ 참가자**: **연결 3단 — 대부분은 붙여넣기가 기본**: ① SFDC 웹에서 계정 페이지 열어 Opportunity·활동 복사·붙여넣기 ② 연결됐으면 계정명으로 자동 조회 ③ 열린 딜 요약 직접 타이핑. → 소스5 STUB 교체 → **5소스 재실행 → 딜까지 붙은 완성 브리핑**
- **✋ STOP**: 5소스 브리핑 + 진행판(소스5 ✅ → 완성) 후 멈춤
- **✅ 체크**: (블록에 준함) "딜 데이터가 채우는 것?" → 단계·금액·Close Date, 파이프라인 위치
- **⚠️ 막히는 곳**: **세일즈 대부분은 SFDC를 웹으로만 쓴다 — 붙여넣기가 정상 경로**임을 미리 못 박기(참가자가 "연결해야 하나" 부담 안 느끼게). **연결 명령을 지어내지 않는다** — 자동 조회는 세팅된 환경만, 없으면 붙여넣기(결과 같음). 고객 실명 금지.
- **🖥 슬라이드**: 카드 ③ 공통 (Gmail·Circleback·Notion·Slack·Salesforce)

## Block 8 — Finalize + 실전 (15분 · 결승선)

- **🎤 강사**: `📖 .../sub-agents`. "지금껏 순차로 하나씩 → 이제 ① 병렬 수집 ② 출력 포맷 완성." **subagent 병렬 = 카페 비유**(직원 1명 순차 vs 4명 병렬). "일부 소스 미연결도 OK, 연결된 소스만으로 돈다." 출력 포맷 3옵션(Markdown 기본 / +Slack / +Notion).
- **⌨️ 참가자**: 출력 형식 택1 → 실행흐름 STUB 완성(소스별 Agent 병렬 호출) → 출력포맷 STUB 완성 → **(결승선) 실제 담당 계정 1개로 "{계정명} 계정 싱크해줘" → 5소스 동시 수집** → 소스별 성공/실패 리포트 확인 → 브리핑 열어 "진짜 나오나?" 확인
- **✋ STOP**: "축하합니다 — my-account-sync 완성" 후 멈춤
- **✅ 체크**: 종합 3연속 — ① 병렬 이유(동시 수집·독립 실행) ② 부분 성공(성공한 소스만으로 브리핑 나옴) ③ 오늘 만든 것 한 문장(계정명→5소스→콜프렙 브리핑)
- **⚠️ 막히는 곳 (핵심)**: **일부 소스 실패가 정상 기대치** → "성공한 소스만으로 충분, 5개 중 2~3개만 돼도 가치 충분, 나머지는 나중에 붙이면 자동으로." 붙여넣기 참가자는 subagent 대신 붙여넣은 내용으로 통합.
- **🖥 슬라이드**: slide-02 지도 카드 ④ (실전 실행 시점에 띄우면 결승선 감각 강화)
