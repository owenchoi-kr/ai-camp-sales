# Day 3 강사 진행 대본

> 진행자용. Day 3의 실제 내용은 `/day3` 스킬이 참가자에게 진행한다. 이 문서는 강사가 각 블록에서 뭘 하고 참가자가 어디서 막히는지 미리 아는 용도다.

## 90분 시간 배분

| 블록 | 주제 | 시간 |
|------|------|------|
| 0 | Recap — Day 2 회고 + 스킬 점검 | 8분 |
| 1 | Clarify — 모호하게 vs 되묻게 (GIGO 체험) | 15분 |
| 2 | Pick — 템플릿 4종 중 택1 | 10분 |
| **3** | **Customize — 슬롯 3곳 내 맥락으로 (산출물)** | **30분** |
| 4 | Run — 실제 딜 1개 실행 + 아쉬운 점 메모 | 15분 |
| 5 | Wrap — 자가진단 + Day 4 예고 | 10분 |

합 88분 + 버퍼 2분. **Block 3(Customize)이 오늘의 목적지, 최대 비중.**

## 공통 STOP PROTOCOL
각 블록 2턴. Phase A(설명 → "직접 실행" 안내 → 멈춤) → 참가자 "완료/다음" → Phase B(퀴즈/체크). Phase A 시작에 `📖 공식 문서: URL` 출력.

---

## Block 0 — Recap (8분)

- **🎤 강사**: Day1(CLAUDE.md=기본값) → Day2(my-account-sync=계정명 넣으면 브리핑) → Day3(그 위에 반복작업 템플릿을 얹는다)를 한 화면으로 되짚음. "세션 새로 켜면 MCP 연결이 풀려 있을 수 있다"를 미리 짚기.
- **⌨️ 참가자**: `/my-account-sync 글로벌 커머스 앱 A사` 또는 "A사 계정 히스토리 뽑아줘" — 어제 스킬이 도는지 점검
- **✋ STOP**: 실행 + (안 되면) 트러블슈팅 안내 후 멈춤. 퀴즈 없음.
- **✅ 체크**: 동작 점검 1문항 (잘 돈다 / 좀 비었다 / 아예 안 돈다 / Day2 못 왔다). 어느 답이든 "오늘은 독립적인 새 스킬을 만든다 → 진도 나간다"로 안심
- **⚠️ 막히는 곳**: MCP 안 불림 → `/mcp` 확인 후 재로그인·재시작. 결과 비면 계정명 철자 확인. **5분 룰** — 안 되면 운영진 사이드 세션, 어제 스킬이 완벽하지 않아도 오늘 진도는 나감.
- **🖥 슬라이드**: slide-01 커버

## Block 1 — Clarify (15분)

- **🎤 강사**: `📖 https://code.claude.com/docs/ko/common-workflows`. GIGO(Garbage In Garbage Out) — "아웃리치 하나 써줘"로 시키면 누구/뭘/톤/길이 모른 채 아무거나 나오고 "아니 그게 아니라"가 3번 반복. 해결책 = Clarify(되묻게 하기). 비유: "신입에게 누구에게·왜·어떤 톤인지 먼저 정해주듯." 관통 메시지 **"일 잘 시키는 사람이 AI도 잘 시킨다."**
- **⌨️ 참가자**: ① `아웃리치 하나 써줘.` (일부러 모호하게, 결과 관찰) ② `아웃리치 쓸 건데, 바로 쓰지 말고 수신자·목적·톤·분량을 먼저 물어봐줘.` ③ 두 결과 비교
- **✋ STOP**: "두 번 다 실행해서 비교해보세요" 문구 후 멈춤
- **✅ 체크**: Q1 소감(정답 없음), Q2 핵심 메시지 → **정답: "일 잘 시키는 사람이 AI도 잘 시킨다"**
- **⚠️ 막히는 곳**: "매번 4가지 묻는 게 번거롭다" → "맞다, 그래서 스킬에 미리 박아둔다(Block 3)"로 다음 블록 연결. 1단계 결과가 그럴싸해 차이 안 느껴지면 → "맥락 0의 기본값이라 일반적"으로 프레이밍.
- **🖥 슬라이드**: slide-02 지도 카드 ①

## Block 2 — Pick (10분)

- **🎤 강사**: `📖 .../skills`. 0.5→1 논리("백지에서 만들면 오래 걸리고 빠진다 → 운영진 0.5 템플릿을 받아 딜 맥락 몇 곳만 고쳐 1로"). 템플릿 4종 각 1줄 소개. **고르는 기준 = "내 주간 루틴에서 지금 손으로 하느라 제일 시간 많이 드는 것" 하나.** 멋져 보이는 것 금지, 오늘 1개만.

  | 템플릿 | 하는 일 | 이런 사람에게 |
  |--------|---------|---------------|
  | call-prep | 계정+미팅 성격 → 콜 프렙 브리핑 | 미팅 준비 잦은 사람 |
  | battlecard | 계정+경쟁 MMP+갱신 → 이관 설득 카드 | 경쟁·갱신 딜 많은 사람 |
  | meeting-followup | 미팅 노트 → 액션+CRM+국/영문 메일 | 팔로업 밀리는 사람 |
  | draft-outreach | 프로스펙트 정보 → 아웃리치 2버전 | 신규 아웃리치 잦은 사람 |

- **⌨️ 참가자**: 터미널 입력 없음. 4종 훑고 "어느 게 내 시간을 제일 먹나" 떠올림
- **✋ STOP**: "정해지면 '선택'" 후 멈춤
- **✅ 체크**: 퀴즈 대신 템플릿 택1(AskUserQuestion 4지선다) → "좋습니다. {템플릿}을 오늘 여러분 것으로 만듭니다."
- **⚠️ 막히는 곳**: 고민할 때 → **"가장 자주 하는 걸 고르면 실패가 없다. 나머지는 2주차 프로젝트에서."**
- **🖥 슬라이드**: slide-02 지도 카드 ②

## Block 3 — Customize (30분 · 오늘의 목적지)

- **🎤 강사**: `📖 .../skills`. "운영진 템플릿은 70% 채워진 0.5판. 비어있는 건 '이게 누구의 스킬인가' = `{{슬롯}}`." 슬롯 3축 예고(①세그먼트 ②톤 ③금지어/용어). 진행 방식: "제가 템플릿을 복사하고 CLAUDE.md를 읽어 슬롯 초안을 제안, 여러분은 다듬기만. 파일은 제가 저장." **이 블록만 Phase A에서 도구 안 씀, Phase B에서 Claude가 적극 실행.**
- **⌨️ 참가자**: 거의 없음 — **복사·수정·저장 전부 Claude가 대신함**(터미널 초보 배려). 참가자는 슬롯 초안 보고 "그대로 좋아요 / 이렇게 고쳐줘"만
- **✋ STOP**: "준비되면 '시작'" 후 멈춤
- **✅ 체크**: 실제 커스터마이징 4단계 — 복사(`cp -r templates/<이름> .claude/skills/<이름>`) → 슬롯 찾기(파일 상단 주석) → CLAUDE.md로 초안 제안(한 슬롯씩) → Edit로 실제 치환·저장
- **⚠️ 막히는 곳 (여기가 핵심)**:
  - **템플릿마다 슬롯이 다르다 — 강사가 미리 알아야 함**:
    - `call-prep`: {{담당_세그먼트_특성}} · {{내_톤}} · {{딜_스테이지_용어}}
    - `battlecard`: {{주력_버티컬}} · {{자주_만나는_반론}} · {{금지_표현}}
    - **`meeting-followup`: {{내_톤_시그니처}} · {{팔로업_기한_기본값}} — 슬롯 2개 (예외!)**
    - `draft-outreach`: {{밸류_프롭_한줄}} · {{타깃_페르소나}} · {{금지어}}
  - 슬롯 못 찾겠다 → 파일 상단 "수정 안내" 주석 다시 읽기
  - CLAUDE.md 얇다 → AskUserQuestion으로 세그먼트·톤·금지어 3개만 빠르게
  - **복사 경로 함정**: 템플릿은 프로젝트 루트 `templates/`에 있음 (`.claude/skills/day3/templates` 아님). `cp -r templates/<이름> .claude/skills/<이름>`을 루트 기준 실행
  - "그대로 좋아요" 하면 초안 확정 — **0.5→1이 목표지 완벽 아님**
- **🖥 슬라이드**: slide-02 지도 카드 ③ (meeting-followup 선택자에겐 "3곳" → "2곳"으로 구두 정정)

## Block 4 — Run (15분)

- **🎤 강사**: `📖 .../slash-commands`. "첫 결과는 대체로 70%. 지금 붙잡고 다듬으면 시간만 녹는다. 오늘 할 일은 ① 실제 딜에서 어떻게 나오나 보기 ② 아쉬운 점 한 줄 남기기." 그 메모는 **내일(Day 4) 까다로운 프로스펙트 채점의 재료**가 됨.
- **⌨️ 참가자**: 진행 중인 딜 1개(실명 부담되면 익명) → `/<내 스킬> A사` 또는 자연어 실행 → 결과를 **고치지 않고** 관찰(내 톤·세그먼트 묻어나나 / 금지어 피했나 / 제일 아쉬운 곳 1곳)
- **✋ STOP**: "실제 딜 1개로 실행하고 아쉬운 점 1개를 정해보세요" 후 멈춤
- **✅ 체크**: 소감(정답 없음) + **아쉬운 점 1개를 한 줄 메모로**. 원하면 Claude가 `NOTES.md`로 저장(강요 금지). "이 한 줄이 내일 Day 4의 출발점."
- **⚠️ 막히는 곳**: 결과를 그 자리에서 고치려 함 → "관찰만, 아쉬운 점은 딱 1개". 스킬 안 불림 → SKILL.md description("언제 쓰는 스킬인가") 한 줄 확인
- **🖥 슬라이드**: slide-02 지도 카드 ④

## Block 5 — Wrap (10분)

- **🎤 강사**: 오늘 바뀐 3가지 — ① "바로 시켰다" → "묻고 시킨다"(Clarify) ② "매번 백지" → "템플릿을 내 것으로"(0.5→1) ③ "일반적 결과" → "내 톤·맥락 묻어남". 산출물: 동작하는 스킬 1개 + 아쉬운 점 메모 1줄. **"차이를 만든 건 AI가 아니라 잘 시킨 나."**
- **⌨️ 참가자**: "다음" → 자가진단 3문항
- **✋ STOP**: "오늘 정리 확인했으면 '다음'" 후 멈춤
- **✅ 체크**: 자가진단 3문항 — ① `/<이름>`으로 부르면 결과 나오나 ② 왜 AI가 잘 시키는 사람에게 더 잘 반응하나(정답: 입력이 명확할수록 결과가 명확, GIGO) ③ 0.5→1에서 '1'을 만든 건(정답: 슬롯 3곳을 내 맥락으로, 2곳은 그대로)
- **⚠️ 막히는 곳**: 아쉬운 점 메모 못 남긴 참가자 → "괜찮다, Day 4 첫 순서가 그걸 함께 뽑는 시간. 오늘 '돌아가는 스킬 1개'면 충분."
- **🖥 슬라이드**: slide-03 마무리
