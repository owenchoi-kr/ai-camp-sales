- **⌨️ 참가자**: 어제까지 산출물 3개 확인 — CLAUDE.md · my-account-sync · 커스터마이징한 스킬 + 아쉬운 점 메모

## 강의 A — 팩트체크 세 가지 마음가짐 (5분)

- **🎤 강사**: slide-03. 리드 스토리 — **"이 6단계는 사내에서 실제로 겪은 사고에서 나왔다. AI가 지어낸 고객 사례를 미팅에 들고 갔다가 반박당한 뒤 이 규칙이 생겼다."** 세 마음가짐: ① AI는 정답 기계가 아니라 **초안 기계** ② **검증은 내 일** (맞나 확인이 아니라 "어디가 틀렸나 찾기" → 이게 곧 실습으로 이어지는 다리) ③ **빈칸을 두려워 않는다** ("모름"이 지어낸 사실보다 프로답다). 습관 3개: "출처?" 되묻기 / 틀렸다고 가정하고 찾기 / 확인 못 하는 건 미팅에 안 들고 간다.
- **✅ 구두 체크**: "AI가 준 건 결과인가 초안인가?" (정답: 초안)
- **⚠️ 막히는 곳**: "AI 못 믿는다는 얘기냐"로 오해 → **"AI를 못 믿어서가 아니라, 제대로 쓰기 위한 것."**
- **🖥 슬라이드**: slide-03

## 강의 B — 팩트체크 6단계 (8분)

- **🎤 강사**: slide-04로 6단계 하나씩 — ① 출처를 함께 남긴다(없으면 [가설]) ② 1차 출처만 인정(공식·보도·IR, 블로그 요약 X) ③ 단정하지 않는다 ④ 검증한 사례만 인용(상대가 그 회사 담당자를 알 수도) ⑤ 없으면 없다고 쓴다 ⑥ 보내기 전 눈으로(회사명·이름·숫자 — "어트리뷰션→어트리모션" 오인식 사례로 각인). **핵심 프레임: "우리가 만든 스킬은 1·2·3·5를 자동으로 지킨다. 4·6은 사람이 한다."**
- **✅ 구두 체크**: "스킬이 자동으로 못 지켜주는 단계 2개는?" (정답: ④ 검증한 사례, ⑥ 보내기 전 눈으로)
- **⚠️ 막히는 곳**: "스킬 결과 그냥 믿어도 되나" → 4·6은 여전히 사람 최종 확인
- **🖥 슬라이드**: slide-04

## 강의 C — 의사결정 로그 루프 (7분)

- **🎤 강사**: slide-05. 프레임 — **팩트체크는 "쓰레기 안 만들기(방어)", 의사결정 로그는 "실력 매주 올리기(공격)."** 시킬 때마다 3가지 남긴다(뭘 시켰나·왜·결과). 원칙 3개: ① **결과 알기 전에** 근거·확신도 먼저 적기(안 그러면 "난 될 줄 알았어"로 오염) ② 기준 먼저 정하고 반복("120자·CTA 하나·자랑 금지") ③ 잘된 사례를 데이터로 복제 (Gong 성사 통화 말하기 43%). 실천법: 주 1회 5분 회고 → 잘된 프롬프트는 템플릿에, 틀린 패턴은 규칙에. **Build → Use → Notice → Fix.**
- **✅ 구두 체크**: "근거·확신도는 결과 전에 적나 후에 적나?" (정답: 전에)
- **⚠️ 막히는 곳**: Gong 43% 수치 인용하려는 참가자 → "인용 시 팩트체크 6단계 적용, 원문 확인, 확신 없으면 [가설]"
- **🖥 슬라이드**: slide-05

## 실습 1 — Verify: 까다로운 프로스펙트 검증 (25분 ⭐ 핵심)

- **🎤 강사 (Phase A)**: `📖 .../sub-agents`. **Generator ≠ Evaluator** — 만든 사람은 빈칸도 "당연히 알겠지" 넘긴다. 그래서 내 맥락을 안 물려받은 **Subagent(별도 프로세스)**를 띄워 채점. 페르소나 3종: ① 바쁜 그로스 리드(첫 3초) → 아웃리치 ② 회의적 CFO형(ROI 숫자) → 업셀 ③ 이관 피로 담당자(전환 리스크) → 배틀카드. 한 사이클: 결과물 → 채점 → 딱 1곳만 고침 → 재채점. **★ 운영진이 먼저 Subagent 채점을 시연하고 참가자가 따라 함.**
- **⌨️ 참가자**: 채점받을 결과물 1개 + 페르소나 1개 고르기 → 붙여넣기 → "딱 하나 고칠 것" **1개만** 반영 후 재제출
- **✋ STOP**: "위 형식으로 결과물을 붙여넣어 주세요" 후 멈춤. **Phase A에서 Task·AskUserQuestion 금지.**
- **✅ 체크**: ① "한 곳만 고쳤는데 반응이 달라졌나?" ② "왜 내가 아니라 Subagent에게 채점시켰나?"(정답: 내 맥락을 안 물려받아 "당연히 알겠지"가 없어서)
- **⚠️ 막히는 곳 (★ 운영진 시연 포인트)**:
  - **메인 Claude가 직접 채점 = 최대 실패.** 반드시 Task 도구로 Subagent를 띄워야 별도 컨텍스트 채점이 됨 — 운영진 시연으로 각인. subagent_type은 general-purpose
  - 채점이 후하게 나옴 → 페르소나 규칙 강조("첫 초안이면 5~7점 정상, 만점 거의 없음")
  - 참가자가 여러 곳 한꺼번에 고침 → "딱 하나만" 재안내
  - 재채점 때 점수 안 변함 → "고친 지점이 반응을 바꿨는지 1줄로 짚기", 안 바뀌면 새 '딱 하나' or 다른 페르소나로 1회 더
- **🖥 슬라이드**: 없음(실습). 강의 A의 "검증은 내 일 / 남의 눈을 빌려야 허점이 보인다"가 이 블록의 토대 — 강사가 명시적으로 콜백

## 실습 2 — Routine: 주간 루틴 조립 (12분)

- **🎤 강사 (Phase A)**: `📖 .../skills`. **조립 = 스킬을 순서로 엮기.** 예시 루틴: ① my-account-sync(이번 주 미팅 계정) → ② call-prep(콜프렙) → ③ meeting-followup(팔로업). 핵심 = **트리거(언제) + 순서(무엇을 어떤 차례로).** 검증도 루틴에 끼우기("쓰기 → 채점 → 1곳 고치기 → 보내기").
- **⌨️ 참가자**: 내 월요일 아침 루틴을 **딱 3줄**로 적기 (스킬 2개뿐이면 2줄)
- **✋ STOP**: "내 루틴을 3줄로 적어주세요 → CLAUDE.md에 넣어드립니다" 후 멈춤. Phase A에서 Edit·AskUserQuestion 금지.
- **✅ 체크**: "이 루틴, 다음 주 월요일에 실제로 돌릴 것 같나요?"
- **⚠️ 막히는 곳**: **트리거(요일·시각) 누락** → "이 루틴 주로 언제 도나요?" 한 번 묻기(트리거 없으면 습관이 안 됨). 참가자 루틴 과하게 교정 금지(정답 루틴 없음). Phase B에서 CLAUDE.md 끝에 "## 내 주간 루틴" 섹션 추가
- **🖥 슬라이드**: 없음(실습). 강의 C의 "잘된 걸 CLAUDE.md에 박는다"의 실천

## 실습 3 — Graduate: 내 것으로 가져가기 (10분 · 캠프의 마침표)

- **🎤 강사 (Phase A)**: "지금까지 만든 CLAUDE.md·스킬은 이 캠프 폴더 안에서만 돈다. 2주차엔 네 **실제 업무**에 적용해야 하는데, 그러려면 전역(`~/.claude/`)으로 옮겨야 어느 폴더에서든 된다." 프레이밍: **"캠프 폴더 = 연습장. 검증 끝난 것만 골라 실제 환경으로 승격한다."** Day 4 검증 원리(검증된 것만 믿는다)가 여기서 "검증된 것만 가져간다"로 이어진다.
- **⌨️ 참가자**: 가져갈 스킬 고르기 → (Claude가 대신) 전역 `~/.claude/skills/`로 복사 + CLAUDE.md 안전 병합 → 다른 폴더에서 작동 확인
- **✋ STOP**: "가져갈 것을 훑어보라" 후 멈춤
- **✅ 체크**: "이제 어느 폴더에서 /call-prep이 되나?"(정답: 어디서든) · "2주차 프로젝트는 어디서 하나?"(정답: 내 실제 업무 폴더)
- **⚠️ 막히는 곳 (가장 중요)**: **전역 `~/.claude/CLAUDE.md`를 절대 덮어쓰지 않는다.** 이미 있으면 "그 아래에 이어붙일까요?" 물어본 뒤 append만, 없으면 복사. 비개발자가 개인 설정을 날리면 복구가 어렵다 — Claude가 대신 처리하되 이 분기를 반드시 지킨다.
- **🖥 슬라이드**: 없음(실습). slide-02 지도 카드 ④ "조립·가져가기·프로젝트"의 가운데

## 실습 4 — Scoping: 2주차 프로젝트 (15분 · 산출물)

- **🎤 강사 (Phase A)**: `📖 .../skills`. **인터뷰 블록.** 2주차 = 내 실제 파이프라인 1개에 적용, 기준은 "매주 반복되는 시간 낭비를 없앨 것". `docs/project-menu.md`의 메뉴 6개를 **요약 없이 그대로** 제시. 없으면 자기 고통점으로도 OK. 인터뷰 4가지 예고: 없앨 반복 작업 / 완성 기준 / 범위 밖 / 쇼케이스 한 장.
- **⌨️ 참가자**: 메뉴 택1 or "내 고통점으로" → "시작" → 한 주제씩 인터뷰 → my-project.md 확정
- **✋ STOP**: "하나 고르거나 '내 고통점으로'라고 알려주세요. 준비되면 '시작'" 후 멈춤. Phase A에서 AskUserQuestion·Write 금지.
- **✅ 체크**: "방금 만든 my-project.md, 지금 상태 어떤가요?"(범위 잡혔다 / 넓다 / 저장 안 됨)
- **⚠️ 막히는 곳**: 4개 한꺼번에 쏟음 → 한 주제씩. **완성 기준 모호("잘 되면")** → 측정 가능하게("계정명 입력 → 3분 안에 브리핑"), 실제 딜 1개 적용을 반드시 포함. 범위 욕심 → "자동 발송·일반화는 범위 밖으로 빼자". **파일 저장 안 함 = 실패** → Write로 `ai-camp-sales/my-project.md` 실제 저장, 있으면 덮어쓰지 말고 물어보기
- **🖥 슬라이드**: 없음(실습). slide-02 "오늘 손에 남는 것 = 2주차 프로젝트 계획 1장"의 실현 지점

## 마무리 — Wrap (8분)

- **🎤 강사 (Phase A)**: slide-06로 4일 산출물 체크리스트 — ☑ CLAUDE.md(+주간 루틴) ☑ my-account-sync ☑ 내 템플릿 스킬 ☑ my-project.md. 4일 흐름 한 줄(프로필→도구로 맥락→커스터마이징→검증·조립·스코핑). 오늘 얻은 습관 2개: **검증**(다른 눈으로 채점) + **조립**(스킬을 루틴으로).
- **⌨️ 참가자**: 산출물 체크 → "다음" → 자가진단 3문항
- **✋ STOP**: "4일 산출물 확인했으면 '다음'" 후 멈춤
- **✅ 체크**: ① 왜 다른 관점에게 채점시켰나(Generator≠Evaluator) ② CLAUDE.md에 주간 루틴이 **트리거와 함께** 있나 ③ my-project.md 범위가 2주 안에 끝낼 만큼 좁나
- **⚠️ 막히는 곳**: 산출물 일부 비어도 OK → "2주차에 채운다, 중요한 건 my-project.md 하나를 실제 딜에 적용". 마지막 메시지: **"새로 대단한 걸 만들지 마세요. 매주 반복되던 시간 낭비 하나를 실제 딜 하나에서 없애면 됩니다."**
- **🖥 슬라이드**: slide-06
