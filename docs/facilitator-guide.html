Day 1 강사 진행 대본

강의 준비할 때 펼쳐놓고 보는 문서다. 참가자용이 아니라 진행자(현종님·예찬님)용이다. Day 1의 실제 내용은 /day1 스킬이 참가자에게 진행한다. 이 문서는 강사가 "각 블록에서 뭘 하고, 참가자가 어디서 막히는지"를 미리 아는 용도다.

전 블록 공통 — STOP PROTOCOL

각 블록은 2턴이다. 강사는 이 리듬을 알고 있어야 참가자가 "다음"을 언제 치는지 안다.

  • Phase A: Claude가 설명(EXPLAIN) → "지금 직접 실행하세요"(EXECUTE) 안내 → 멈춤
  • 참가자가 터미널에서 실행 → "완료" 또는 "다음" 입력
  • Phase B: 퀴즈/체크 → 다음 블록

Phase A에서는 퀴즈·질문을 하지 않는다. 각 블록 시작에 📖 공식 문서: URL이 그대로 출력된다.

Block 0 — Setup (5분)

  • 🎤 강사: "지금 이 터미널 화면이 Claude Code다. 오늘 여기 하나로 다 한다"를 짚어준다. 설치는 어젯밤 숙제(README Step 0)로 끝난 전제.
  • ⌨️ 참가자: ① /config → Output style → Explanatory ② 안녕, 나는 [이름]이고 세일즈야. 인사해줘 ③ 지금 내가 어느 폴더에서 너를 실행하고 있어? → ai-camp-sales 나오면 정상
  • ✋ STOP: 폴더 확인 명령 안내 후 멈춤. 퀴즈 없음.
  • ⚠️ 막히는 곳: 설치·로그인 실패 → claude.ai에 "Claude Code 설치하는 방법 알려줘" 입력해 환경별 안내. 90%는 재로그인 또는 재시작으로 해결. 폴더가 ai-camp-sales가 아니면 이동 후 재실행. 에디터(Cursor/VSCode)는 오늘 불필요.
  • 🖥 슬라이드: slide-01 커버로 오프닝 → slide-02 지도로 오늘 4블록 먼저 보여주기

Block 1 — Experience (12분)

  • 🎤 강사: 핵심 메시지 — "맥락 없이도 이 정도가 나온다. 4일 뒤엔 여러분 도구가 연결된 버전이 된다." Before/After(지금=스레드·노션·메일 수동 확인 → 4일 뒤=계정명 하나로 자동 브리핑). "지금은 도구 0개, 그래도 어디까지 나오나 본다."
  • ⌨️ 참가자: ① 다음 주 실제 미팅 1개 선택(실명 부담되면 "글로벌 커머스 앱 A사"로 익명화) ② 틀에 붙여넣기(계정/미팅 성격/참석자/아젠다/아는 배경 — 아는 만큼만) ③ 나온 브리핑을 고치지 말고 관찰
  • ✋ STOP: 붙여넣기+관찰 안내 후 멈춤
  • ✅ 체크: 정답 없는 소감. "도구 0개여도 구조는 나온다 / 내 딜의 진짜 맥락은 비었다 = 4일의 과제"를 먼저 정리해준 뒤 소감 질문

Block 2 — Why (10분)
