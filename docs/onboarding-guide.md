# 온보딩 — 설치·시작을 어떻게 설명하나 (운영자용)

> 킥오프에서 참가자에게 "설치 → 캠프 받기 → 나중에 가져가기"를 어떻게 설명할지 정리한 진행 가이드다. 참가자는 대부분 터미널을 처음 쓰는 세일즈다.

## 한 문장으로: "연습장 → 진짜 사무실"

git·전역·로컬 같은 단어는 한 번도 쓰지 않는다. 개념을 다 빼고 **비유 하나**로 묶는다.

> 캠프 폴더 = **여러분의 연습장.**
> 여기서 막 만들어보고, 마지막 날 검증된 것만 골라 **진짜 사무실(실제 업무 환경)로 이사**한다.

이 비유가 캠프 전체를 관통한다 — 1주차는 연습장에서 만들고, Day 4에서 검증하고, 2주차엔 진짜 사무실에서 실제 딜에 쓴다.

---

## 참가자에게 할 말 (그대로 읽어도 되는 스크립트)

> "오늘 여러분은 폴더 하나를 받습니다. 이게 여러분의 **AI 캠프 연습장**이에요.
>
> - 매일 이 안에서 Claude를 열고 `/day1`부터 `/day4`까지 합니다.
> - 여기서 만드는 CLAUDE.md, 스킬은 다 이 연습장 안에만 쌓여요. **막 해봐도 여러분 회사 컴퓨터 설정은 안 건드립니다.** 안전하니까 마음껏 실수하세요.
> - 마지막 날, 그중에 **진짜 쓸 만한 것만 골라서 여러분의 실제 작업 환경으로 옮깁니다.** 그때부터 어느 폴더에서든 써요.
> - 여러분이 기억할 건 딱 하나입니다: **매일 이 폴더에서 Claude 열기.**"

---

## 3가지 설명 원칙

1. **왜 먼저, 명령어는 복붙** — "이 3줄을 복사해서 붙여넣으세요"라고만 한다. 명령어 뜻은 설명하지 않는다. 외우게 하지 않는다.
2. **딱 하나만 기억시키기** — "매일 이 폴더에서 claude 연다." 나머지는 `/dayN` 스킬이 알아서 안내한다.
3. **안전하다는 걸 강조** — 비개발자의 진짜 두려움은 "내가 뭘 잘못 건드릴까 봐"다. "연습장이라 회사 설정 안 건드린다"가 그 두려움을 없앤다.

---

## 진짜 함정: 혼자 설치해오게 하지 마라

가장 위험한 가정은 "참가자가 설치·clone을 사전에 혼자 해올 것"이다. 터미널 초보가 이걸 혼자 성공할 확률은 낮다 (권한 에러, 경로 문제, 로그인 실패...). 그럼 Day 1 시작할 때 절반이 "저 안 돼요"에서 멈춘다.

**→ 킥오프 첫 15~20분을 "다 같이 설치" 세션으로 잡는다.** 설명을 아무리 잘 써도 옆에서 봐주는 것만 못하다. 사전 안내는 "미리 해보고 막히면 표시해두세요" 정도로만.

### 다 같이 설치 진행 순서 (15~20분)

1. **화면 공유로 운영진이 먼저 시연** — 3줄을 한 줄씩 천천히.
2. 참가자가 각자 복붙 → 실행. 운영진·짝꿍이 돌아다니며 막힌 사람 표시 확인.
3. "터미널에서 `claude` 입력 → 인사 나오면 성공" 을 전원 확인.
4. `/day1` 첫 화면까지 뜨면 온보딩 끝. 여기서부터 스킬이 진행한다.

---

## 참가자가 복붙할 3줄

이미 Claude Code를 설치한 사람은 1번(설치)을 건너뛰고 2번부터.

```bash
# 1. Claude Code 설치 (이미 있으면 건너뛰기)
curl -fsSL https://claude.ai/install.sh | bash

# 2. 캠프 연습장 받기
git clone https://github.com/owenchoi-kr/ai-camp-sales.git
cd ai-camp-sales

# 3. 열기
claude
```

그다음 Claude Code 안에서 `/day1`.

## 자주 막히는 곳 · 대응

| 증상 | 대응 |
|------|------|
| 설치 명령이 에러 | claude.ai 대화창에 "Claude Code 설치하는 방법 알려줘" 입력 → 환경별 안내 |
| 로그인이 안 됨 | 구독 계정(Pro/Max/Team/Enterprise) 확인. 재로그인 |
| `git clone`이 안 됨 | git 미설치일 수 있음. macOS는 `xcode-select --install` 안내. 안 되면 짝꿍 화면으로 대신 진행 후 폴더 공유 |
| clone은 됐는데 `/day1`이 없다 | **`ai-camp-sales` 폴더 안에서** claude를 열었는지 확인. `cd ai-camp-sales` 다시 |
| "이거 회사 설정 건드리는 거 아니에요?" | "연습장이라 안 건드립니다. 마지막 날에만 골라서 옮겨요" — 안심시키기 |

> 핵심 반복: 온보딩에서 참가자가 가져갈 메시지는 딱 두 개다. **① 매일 이 폴더에서 claude 연다 ② 여긴 안전한 연습장이다.**
