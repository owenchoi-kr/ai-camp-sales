> Day 4에서 하나를 고르고 스코핑 1장을 쓴 뒤 2주차에 만든다. 기준: **내 실제 딜/루틴에서 시간을 아낄 것.** 새로운 걸 만들기보다 매주 반복되는 것을 없애는 게 목표.

## 스코핑 1장 템플릿 (Day 4에서 작성)

```markdown
# 프로젝트: {이름}

## 없애려는 반복 작업
- 무엇을: (예: 미팅 전 30분씩 걸리는 계정 히스토리 조사)
- 얼마나 자주: (예: 주 5회)
- 지금 걸리는 시간: (예: 회당 30분)

## 완성 기준 (이걸 보면 "됐다")
- [ ] (예: 계정명 입력 → 3분 안에 브리핑 문서)
- [ ] 실제 딜 1개에 적용해서 다음 미팅에 썼다

## 범위 밖 (이번엔 안 하는 것)
- (예: 자동 발송, 팀 공유용 일반화)

## 쇼케이스 한 장
- Before: 걸리던 시간 → After: 지금 걸리는 시간
```

## 운영 규칙

- 2주차 수요일 중간 체크인 1회 (Slack 스레드로 진행 상황 + 막힌 곳)
- 막히면 그날 물어보기 — 금요일에 몰아서 오면 못 도와준다
- 쇼케이스 발표는 3분, before/after 1장. 데모 라이브면 가산점
- 잘 된 프로젝트는 캠프 후 abot 스킬 승격 후보로 제안
