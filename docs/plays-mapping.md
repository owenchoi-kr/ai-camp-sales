# 관찰된 세일즈 플레이 8개 → 커리큘럼 매핑

> 세일즈팀 Slack·미팅노트에서 실제 관찰된 반복 수작업 8개. 커리큘럼의 실습 재료와 2주차 프로젝트 메뉴는 전부 여기서 나왔다.
> 원칙: **캠프 1주차는 "개인이 Claude Code로 직접"(A·C 유형), 팀 자동화(B 유형, abot·스케줄)는 2주차 프로젝트 또는 캠프 이후 abot 스킬 제안으로.**

## 1주차 커리큘럼에 직접 들어가는 것

### 플레이 1 — 콜 프렙 브리핑 (캠프의 척추)

세 번에 걸쳐 심화된다:

- **Day 1**: 운영진이 실제 미팅 하나로 라이브 데모 — "이게 캠프 끝나면 여러분 것이 된다"
- **Day 2**: 참가자가 `my-account-sync` 스킬을 직접 빌드 — gws(Gmail) · Circleback · Notion · Slack 4소스에서 계정 히스토리를 병렬 수집
- **Day 3**: `templates/call-prep`을 내 세그먼트·내 딜 스테이지에 맞게 0.5→1 커스터마이징

원본 플레이의 4개 소스(SFDC Opportunity 히스토리 · 과거 Slack 논의 · 경쟁사 갱신시기 · 유사 버티컬 사례) 중 SFDC는 참가자 개별 인증 이슈가 있어 **캠프에서는 gws·Circleback·Notion·Slack 4소스로 시작**하고, SFDC 연결은 프로젝트 메뉴에서 도전 과제로.

### 플레이 2 — 배틀카드

`templates/battlecard`: 계정명 + 경쟁사 + 갱신 시기를 입력하면 버티컬 맞춤 이관 설득 포인트를 생성. 과거 이관 성공 사례(Slack 검색)를 근거로 붙인다.

### 플레이 7 — 미팅 팔로업

`templates/meeting-followup`: Circleback 미팅노트 → ①핵심 액션아이템 ②SFDC 업데이트 제안 ③고객 팔로업 메일 초안(국/영문). 발송은 gws draft로 (바로 발송하지 않고 초안 생성까지 — 사람이 확인 후 발송).

## 2주차 프로젝트로 가는 것 (플레이 3·4·5·6·8)

팀 봇(abot)·스케줄 자동화가 필요한 B 유형은 개인 캠프 범위를 넘는다. 대신 2주차에 **"내 것부터 만드는" 개인 버전**으로 스코핑한다 → [project-menu.md](project-menu.md). 캠프 후 잘 된 것을 abot 스킬로 승격하는 경로를 쇼케이스에서 제안.

## 캠프에서 다루지 않는 것 (명시적 제외)

- abot 서버사이드 스킬 개발 자체 (참가자는 "abot에 제안할 스펙"까지만)
- SFDC 쓰기 작업 자동화 (조회는 프로젝트에서 가능, 쓰기는 RevOps 협의 필요)
- 고객 메일 자동 발송 (초안 생성까지만 — 발송은 항상 사람이)
